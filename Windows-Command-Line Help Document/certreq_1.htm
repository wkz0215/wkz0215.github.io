certreq

  • 项目
  • 09/21/2022

本文内容

certreq 命令可用于从证书颁发机构请求证书 (CA) ，从 CA 检索对以前的请求的响应，从 .inf 文件创建新请求，以接受和安装对请求的响应，以从现有 CA 证书或请求构造交叉认证或限定的次序请求， 并签署交叉认证或限定的下级请求。

重要

certreq 命令的早期版本可能无法提供此处介绍的所有选项。 若要查看基于特定版本的 certreq 支持的选项，请运行命令行帮助选项 certreq -v -?。

certreq 命令不支持在 CEP/CES 环境中基于密钥证明模板创建新的证书请求。

警告

本主题的内容基于 Windows 服务器的默认设置;例如，将密钥长度设置为 2048，选择 Microsoft 软件密钥存储提供程序作为 CSP，并使用安全哈希算法 1 (SHA1) 。 根据公司安全策略的要求评估这些选择。

语法

certreq [-submit] [options] [requestfilein [certfileout [certchainfileout [fullresponsefileOut]]]]
certreq -retrieve [options] requestid [certfileout [certchainfileout [fullresponsefileOut]]]
certreq -new [options] [policyfilein [requestfileout]]
certreq -accept [options] [certchainfilein | fullresponsefilein | certfilein]
certreq -sign [options] [requestfilein [requestfileout]]
certreq –enroll [options] templatename
certreq –enroll –cert certId [options] renew [reusekeys]

参数

参数 说明
-submit 向证书颁发机构提交请求。
-retrieve <requestid> 从证书颁发机构检索对以前的请求的响应。
-new 从 .inf 文件创建新请求。
-accept 接受并安装对证书请求的响应。
-policy 设置请求的策略。
-sign 签署交叉认证或限定的次序号请求。
-enroll 注册或续订证书。
-? 显示 certreq 语法、选项和说明的列表。
<parameter> -? 显示指定参数的帮助。
-v -？ 显示 certreq 语法、选项和说明的详细列表。

示例

certreq -submit

提交简单的证书请求：

certreq –submit certrequest.req certnew.cer certnew.pfx

注解

  • 这是默认certreq.exe参数。 如果未在命令行提示符下指定任何选项，certreq.exe尝试向证书颁发机构提交证书请求。 使用 –submit 选项时，必须指定证书请求文件。 如果省略此参数，将显示一个常见的 “文件打开 ”窗口，让你选择相应的证书请求文件。

  • 若要通过指定 SAN 属性来请求证书，请参阅 如何使用 certreq.exe 实用工具创建和提交 Microsoft 知识库文章的证书请求部分，931351 如何将使用者可选名称添加到安全 LDAP 证书。

certreq -retrieve

若要检索证书 ID 20 并创建名为 MyCertificate 的证书文件 (.cer) ：

certreq -retrieve 20 MyCertificate.cer

注解

  • 使用 certreq -retrieve requestid 在证书颁发机构颁发证书后检索证书。 requestid PKC 可以是带 0x 前缀的十进制或十六进制，可以是没有 0x 前缀的证书序列号。 还可以使用它来检索证书颁发机构颁发的任何证书，包括已吊销或过期的证书，而不考虑证书的请求是否处于挂起状态。

  • 如果向证书颁发机构提交请求，证书颁发机构的策略模块可能会使请求处于挂起状态，并将 requestid 返回到 certreq 调用方进行显示。 最终，证书颁发机构的管理员将颁发证书或拒绝请求。

certreq -new

示例

若要在 记事本 中创建策略文件 (.inf) ，并将其保存为 requestconfig.inf：

[NewRequest]
Subject = CN=<FQDN of computer you are creating the certificate>
Exportable = TRUE
KeyLength = 2048
KeySpec = 1
KeyUsage = 0xf0
MachineKeySet = TRUE
[RequestAttributes]
CertificateTemplate=WebServer
[Extensions]

certreq –new requestconfig.inf certrequest.req

_continue_ = %szOID_PKIX_KP_CLIENT_AUTH%

certreq -accept

该 –accept 参数将先前生成的私钥与颁发的证书链接在一起，并从请求证书的系统中删除挂起的证书请求， (如果存在匹配的请求) 。

若要手动接受证书，请执行以下操作：

certreq -accept certnew.cer

警告

使用-accept参数和-user–machine选项指示安装证书是否应在用户或计算机上下文中安装。 如果任一上下文中有一个与所安装的公钥匹配的未完成请求，则不需要这些选项。 如果没有未完成的请求，则必须指定其中一个请求。

certreq -policy

policy.inf 文件是一个配置文件，用于定义在定义限定的次序号时应用于 CA 认证的约束。

若要生成跨证书请求，请执行以下操作：

certreq -policy certsrv.req policy.inf newcertsrv.req

无需 certreq -policy 任何其他参数即可打开对话框窗口，使你可以选择请求的文件 (.req、.cmc、.txt、.der、.cer 或 .crt) 。 选择请求的文件并单击“ 打开”后，将打开另一个对话框窗口，允许你选择 policy.inf 文件。

示例

在 CAPolicy.inf 语法中查找 policy.inf 文件的示例。

certreq -sign

若要创建新的证书请求，请对其进行签名并提交：

certreq -new policyfile.inf myrequest.req
certreq -sign myrequest.req myrequest.req
certreq -submit myrequest_sign.req myrequest_cert.cer

注解

  • 如果不使用 certreq -sign 任何其他参数，它将打开对话框窗口，以便你可以选择请求的文件 (req、cmc、txt、der、cer 或 crt) 。

  • 对限定的次序号请求进行签名可能需要Enterprise管理员凭据。 这是颁发合格次序签名证书的最佳做法。

  • 用于对限定的次序号请求进行签名的证书使用限定的次序模板。 Enterprise管理员必须对请求进行签名或向用户授予对证书签名的个人权限。

  • 可能需要在 CMC 请求后让其他人员对 CMC 请求进行签名。 这取决于与限定的次序关联的保证级别。

  • 如果要安装的合格从属 CA 的父 CA 处于脱机状态，则必须从脱机父级获取限定的从属 CA 的 CA 证书。 如果父 CA 处于联机状态，请在 证书服务安装 向导中为限定的从属 CA 指定 CA 证书。

certreq -enroll

可以使用此注释来注册或续订证书。

示例

若要使用 WebServer 模板注册证书，并使用 U/I 选择策略服务器：

certreq -enroll –machine –policyserver * WebServer

若要使用序列号续订证书，请执行以下操作：

certreq –enroll -machine –cert 61 2d 3c fe 00 00 00 00 00 05 renew

只能续订有效的证书。 无法续订过期的证书，并且必须替换为新证书。
