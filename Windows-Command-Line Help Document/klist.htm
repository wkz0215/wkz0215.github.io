klist

  • 项目
  • 09/24/2022

本文内容

显示当前缓存的 Kerberos 票证的列表。

重要

必须至少是 域管理员或等效者才能运行此命令的所有参数。

语法

klist [-lh <logonID.highpart>] [-li <logonID.lowpart>] tickets | tgt | purge | sessions | kcd_cache | get | add_bind | query_bind | purge_bind

参数

参数 说明
-lh 表示用户本地唯一标识符的较高部分（以十六进制 (LUID) 表示）。 如果 –lh 和 –li 都不存在，则命令默认为当前登录的用户的 LUID。
-li 表示用户本地唯一标识符的低部分 (LUID) 以十六进制表示。 如果 –lh 和 –li 都不存在，则命令默认为当前登录的用户的 LUID。
票 列出当前缓存的票证授予票证 (TGT) 以及指定登录会话的服务票证。 这是默认选项。
tgt 显示初始 Kerberos TGT。
清除 允许删除指定登录会话的所有票证。
会话 显示此计算机上登录会话的列表。
kcd_cache 显示 Kerberos 约束委派缓存信息。
get 允许向 SPN 服务主体名称指定的目标计算机请求票证 (SPN) 。
add_bind 允许你为 Kerberos 身份验证指定首选域控制器。
query_bind 显示 Kerberos 已联系的每个域的缓存首选域控制器列表。
purge_bind 删除指定域的缓存首选域控制器。
kdcoptions 显示 RFC 4120 密钥发行中心 (KDC) 选项。
/? 显示此命令的帮助。

注解

  • 如果未提供参数， klist 将检索当前登录用户的所有票证。

  • 参数显示以下信息：

    • tickets - 列出自登录以来已经过身份验证的服务当前缓存的票证。 显示所有缓存票证的以下属性：

      • LogonID： LUID。

      • 客户： 客户端名称和客户端域名的串联。

      • 服务器： 服务名称和服务的域名的串联。

      • KerbTicket 加密类型： 用于加密 Kerberos 票证的加密类型。

      • 票证标志： Kerberos 票证标志。

      • 开始时间： 票证有效的时间。

      • 结束时间： 票证不再有效的时间。 当票证超过此时间时，它不再可用于向服务进行身份验证或用于续订。

      • 续订时间： 需要新的初始身份验证的时间。

      • 会话密钥类型： 用于会话密钥的加密算法。

    • tgt - 列出当前缓存票证的初始 Kerberos TGT 和以下属性：

      • LogonID： 以十六进制标识。

      • ServiceName： krbtgt

      • TargetName ： krbtgt

      • DomainName： 用于问题 TGT 的域的名称。

      • TargetDomainName： TGT 颁发给的域。

      • AltTargetDomainName： TGT 颁发给的域。

      • 票证标志： 地址和目标操作和类型。

      • 会话密钥： 密钥长度和加密算法。

      • StartTime： 请求票证的本地计算机时间。

      • EndTime： 票证不再有效的时间。 当票证超过此时间时，它不再可用于向服务进行身份验证。

      • RenewUntil： 票证续订的截止时间。

      • TimeSkew： KDC 密钥发行中心 (的) 。

      • EncodedTicket： 编码票证。

    • purge - 允许删除特定票证。 清除票证会销毁已缓存的所有票证，因此请谨慎使用此属性。 它可能会阻止你向资源进行身份验证。 如果发生这种情况，必须注销并再次登录。

      • LogonID： 以十六进制标识。

    • sessions - 允许列出并显示此计算机上所有登录会话的信息。

      • LogonID： 如果指定，则仅按给定的值显示登录会话。 如果未指定，则显示此计算机上的所有登录会话。

    • kcd_cache - 允许显示 Kerberos 约束委派缓存信息。

      • LogonID： 如果指定，则按给定的值显示登录会话的缓存信息。 如果未指定，则显示当前用户的登录会话的缓存信息。

    • get - 允许向 SPN 指定的目标请求票证。

      • LogonID： 如果指定，则通过使用给定值的登录会话请求票证。 如果未指定，则使用当前用户的登录会话请求票证。

      • kdcoptions： 请求具有给定 KDC 选项的票证

    • add_bind - 允许为 Kerberos 身份验证指定首选域控制器。

    • query_bind - 允许显示域的缓存首选域控制器。

    • purge_bind - 允许删除域的缓存首选域控制器。

    • kdcoptions - 有关选项及其说明的当前列表，请参阅 RFC 4120。

示例

若要查询 Kerberos 票证缓存以确定是否缺少任何票证、目标服务器或帐户是否出错，或者是否由于事件 ID 27 错误而不支持加密类型，请键入：

klist

klist –li 0x3e7

若要了解计算机上为登录会话缓存的每个票证授予票证的具体信息，请键入：

klist tgt

若要清除 Kerberos 票证缓存，请注销，然后重新登录，键入：

klist purge

klist purge –li 0x3e7

若要诊断登录会话并查找用户或服务的 logonID，请键入：

klist sessions

若要诊断 Kerberos 约束委派失败并查找遇到的最后一个错误，请键入：

klist kcd_cache

若要诊断用户或服务能否获取到服务器的票证，或请求特定 SPN 的票证，请键入：

klist get host/%computername%

若要诊断域控制器之间的复制问题，通常需要客户端计算机以特定域控制器为目标。 若要将客户端计算机定向到特定域控制器，请键入：

klist add_bind CONTOSO KDC.CONTOSO.COM

klist add_bind CONTOSO.COM KDC.CONTOSO.COM

若要查询此计算机最近联系的域控制器，请键入：

klist query_bind

若要重新发现域控制器，或在使用 创建新的域控制器绑定之前刷新缓存 klist add_bind，请键入：

klist purge_bind
