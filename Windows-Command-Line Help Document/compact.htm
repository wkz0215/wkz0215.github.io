compact

  • 项目
  • 09/21/2022

本文内容

显示或更改 NTFS 分区上的文件或目录的压缩。 如果在没有参数的情况下使用， compact 将显示当前目录及其包含的所有文件的压缩状态。

语法

compact [/C | /U] [/S[:dir]] [/A] [/I] [/F] [/Q] [/EXE[:algorithm]] [/CompactOs[:option] [/windir:dir]] [filename [...]]

参数

参数 说明
/c 压缩指定的目录或文件。 将对目录进行标记，以便压缩以后添加的任何文件，除非指定了/EXE 参数。
/U Uncompresses 指定的目录或文件。 已对目录进行标记，因此以后添加的任何文件都不会压缩。 如果指定了/EXE 参数，则仅解压缩作为可执行文件压缩的文件;如果未指定/EXE 参数，则仅对 NTFS 压缩文件进行解压缩。
/s[:<dir>] 对指定目录和所有子目录中的文件执行所选操作。 默认情况下，当前目录 <dir> 用作值。
/a 显示隐藏文件或系统文件。 默认情况下，这些文件不包括在内。
/i 继续执行指定的操作，忽略错误。 默认情况下，此命令在遇到错误时停止。
/f 强制对指定目录或文件进行压缩或解压缩。 默认情况下，会跳过压缩的文件。 如果文件是在系统崩溃中断操作中断的情况下进行部分压缩的，则使用 /f 参数。 若要强制完全压缩文件，请使用 /c 和 /f 参数，并指定部分压缩的文件。
/q 仅报告最重要的信息。
/EXE 对经常读取但不修改的可执行文件使用优化的压缩。 支持的算法包括：
  • XPRESS4K (最快和默认值)
  • XPRESS8K
  • XPRESS16K
  • LZX (最简洁)
/CompactOs 设置或查询系统的压缩状态。 支持的选项包括：
  • 查询 -查询系统的 Compact 状态。
  • 始终 对所有操作系统二进制文件进行压缩，并将系统状态设置为 Compact，这会保持不变，除非管理员对其进行更改。
  • 永远不会 Uncompresses 所有操作系统二进制文件并将系统状态设置为 "非压缩"，除非管理员对其进行更改。
/windir 在查询脱机操作系统时，与 /CompactOs： query 参数一起使用。 指定安装 Windows 的目录。
<filename> 指定模式、文件或目录。 您可以使用多个文件名 * 以及和 * 通配符。
/? 在命令提示符下显示帮助。

注解

  • 此命令是 NTFS 文件系统压缩功能的命令行版本。 目录的压缩状态指示将文件添加到目录时是否自动对文件进行压缩。 设置目录的压缩状态不一定会更改目录中已有文件的压缩状态。

  • 不能使用此命令来读取、写入或装入使用 "磁盘空间管理" 或 DoubleSpace 进行压缩的卷。 还不能使用此命令 (FAT) 或 FAT32 分区压缩文件分配表。

示例

若要设置当前目录、其子目录和现有文件的压缩状态，请键入：

compact /c /s

若要设置当前目录中的文件和子目录的压缩状态，而不改变当前目录本身的压缩状态，请键入：

compact /c /s *.*

若要压缩卷，请在卷的根目录中键入：

compact /c /i /s:\

注意

此示例将 (包含卷上的根目录的所有目录的压缩状态设置) 并压缩卷上的每个文件。 /I参数可防止错误消息中断压缩进程。

若要压缩 \tmp 目录和 \tmp 的所有子目录中具有 .bmp 文件扩展名的所有文件，而不修改目录的压缩属性，请键入：

compact /c /s:\tmp *.bmp

若要强制对在系统崩溃期间部分压缩的文件 zebra.bmp进行完全压缩，请键入：

compact /c /f zebra.bmp

若要从目录 c:\tmp 中删除压缩属性，但不更改该目录中任何文件的压缩状态，请键入：

compact /u c:\tmp
