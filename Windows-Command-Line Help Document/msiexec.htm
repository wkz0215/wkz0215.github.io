msiexec

  • 项目
  • 09/21/2022

本文内容

提供从命令行对安装程序进行安装、修改Windows操作的方式。

安装选项

设置用于启动安装包的安装类型。

语法

msiexec.exe [/i][/a][/j{u|m|/g|/t}][/x] <path_to_package>

参数

参数 说明
/i 指定正常安装。
/a 指定管理安装。
/ju 将产品播发到当前用户。
/jm 向所有用户播发产品。
/j/g 指定播发的包使用的语言标识符。
/j/t 将转换应用于播发的包。
/x 卸载包。
<path_to_package> 指定安装包文件的位置和名称。

示例

若要使用常规安装 example.msi 从 C： 驱动器安装名为example.msi的包，请键入：

msiexec.exe /i "C:\example.msi"

显示选项

你可以根据目标环境配置用户在安装过程中看到什么。 例如，如果要将包分发到所有客户端进行手动安装，则应该具有完整的 UI。 但是，如果使用 组策略部署包，这不需要用户交互，则不应涉及 UI。

语法

msiexec.exe /i <path_to_package> [/quiet][/passive][/q{n|b|r|f}]

参数

参数 说明
<path_to_package> 指定安装包文件的位置和名称。
/quiet 指定静默模式，这意味着无需用户交互。
/passive 指定无人参与模式，这意味着安装只显示进度栏。
/qn 指定在安装过程中没有 UI。
/qn+ 指定在安装过程中没有 UI，但末尾有一个最终对话框。
/qb 指定在安装过程中有一个基本 UI。
/qb+ 指定在安装过程中有一个基本 UI，包括末尾的最终对话框。
/qr 指定在安装过程中减少的 UI 体验。
/qf 指定安装过程中的完整 UI 体验。
注解
  • 如果用户取消了安装，则模式框不会显示。 可以使用 qb+！ 或 qb！+ 隐藏 "取消" 按钮。

示例

若要 在C:\example.msi安装 包，请键入：

msiexec.exe /i "C:\example.msi" /qn

重启选项

如果安装包覆盖文件或尝试更改使用中的文件，可能需要在安装完成之前重新启动。

语法

msiexec.exe /i <path_to_package> [/norestart][/promptrestart][/forcerestart]

参数

参数 说明
<path_to_package> 指定安装包文件的位置和名称。
/norestart 安装完成后，停止设备重启。
/promptrestart 如果需要重新启动，则提示用户。
/forcerestart 安装完成后重启设备。

示例

若要 在C:\example.msi安装 包，请键入：

msiexec.exe /i "C:\example.msi" /norestart

日志记录选项

如果需要调试安装包，可以设置参数以创建包含特定信息的日志文件。

语法

msiexec.exe [/i][/x] <path_to_package> [/L{i|w|e|a|r|u|c|m|o|p|v|x+|!|*}] <path_to_log>

参数

参数 说明
/i 指定正常安装。
/x 卸载包。
<path_to_package> 指定安装包文件的位置和名称。
/li 启用日志记录，并包含输出日志文件中的状态消息。
/lw 启用日志记录，在输出日志文件中包含非严重警告。
/le 启用日志记录，并包含输出日志文件中的所有错误消息。
/la 启用日志记录，并包含有关何时在输出日志文件中启动操作的信息。
/lr 启用日志记录，在输出日志文件中包含特定于操作的记录。
/lu 启用日志记录，在输出日志文件中包含用户请求信息。
/lc 启用日志记录，并包含输出日志文件中的初始 UI 参数。
/lm 启用日志记录，在输出日志文件中包含内存不足或严重退出信息。
/lo 启用日志记录，在输出日志文件中包含磁盘空间不足的消息。
/lp 启用日志记录，在输出日志文件中包含终端属性。
/lp 启用日志记录，在输出日志文件中包含终端属性。
/lv 启用日志记录，在输出日志文件中包含详细输出。
/lp 启用日志记录，在输出日志文件中包含终端属性。
/lx 启用日志记录，在输出日志文件中包含额外的调试信息。
/l+ 启用日志记录，将信息追加到现有日志文件。
/l！ 启用日志记录，将每行刷新到日志文件。
/l* 启用日志记录并记录所有信息， (/lv) 或 /lx (调试) 。
<path_to_logfile> 指定输出日志文件的位置和名称。

示例

若要 在C:\example.msi安装包，请通过常规安装过程以及提供的所有日志记录信息（包括详细输出）以及将输出日志文件存储在 C：\package.log 中，键入：

msiexec.exe /i "C:\example.msi" /L*V "C:\package.log"

更新选项

可以使用安装包应用或删除更新。

语法

msiexec.exe [/p][/update][/uninstall[/package<product_code_of_package>]] <path_to_package>

参数

参数 说明
/p 安装修补程序。 如果要以无提示方式安装，则还必须将 REINSTALLMODE 属性设置为 ecmus ，将"重新安装"设置为 ALL。 否则，修补程序仅更新目标设备上缓存的 MSI。
/update 安装修补程序选项。 如果要应用多个更新，则必须使用分号分隔 (;) 。
/package 安装或配置产品。

示例

msiexec.exe /p "C:\MyPatch.msp"
msiexec.exe /p "C:\MyPatch.msp" /qb REINSTALLMODE="ecmus" REINSTALL="ALL"
msiexec.exe /update "C:\MyPatch.msp"

msiexec.exe /uninstall {1BCBF52C-CD1B-454D-AEF7-852F73967318} /package {AAD3D77A-7476-469F-ADF4-04424124E91D}

其中第一个 GUID 是修补程序 GUID，第二个 GUID 是应用了修补程序的 MSI 产品代码。

修复选项

可以使用此命令修复已安装的包。

语法

msiexec.exe [/f{p|o|e|d|c|a|u|m|s|v}] <product_code>

参数

参数 说明
/fp 如果缺少文件，则修复包。
/fo 如果缺少文件或安装了旧版本，则修复包。
/fe 如果缺少文件，或者安装了相同或较旧版本，则修复包。
/fd 如果缺少文件或安装了其他版本，则修复包。
/fc 如果缺少文件，或者校验和与计算值不匹配，则修复包。
/fa 强制重新安装所有文件。
/fu 修复所有必需的特定于用户的注册表项。
/fm 修复所有必需的特定于计算机的注册表项。
/fs 修复所有现有快捷方式。
/fv 从源运行，然后重新缓存本地包。

示例

若要强制根据要修复的 MSI 产品代码重新安装所有文件，请 键入 {AAD3D77A-7476-469F-ADF4-04424124E91D}：

msiexec.exe /fa {AAD3D77A-7476-469F-ADF4-04424124E91D}

设置公共属性

可以通过此命令设置公共属性。 有关可用属性以及如何设置它们的信息，请参阅 公共属性。
