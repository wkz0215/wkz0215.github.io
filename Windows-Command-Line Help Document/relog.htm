relog

  • 项目
  • 09/24/2022

本文内容

适用范围：Windows Server 2022、Windows Server 2019、Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

将性能计数器从性能计数器日志中提取到其他格式，例如用于制表符分隔文本) 的文本 TSV (、逗号分隔文本) 、二进制 BIN 或SQL的文本 CSV (。

注意

有关将重新记录合并到 Windows Management Instrumentation (WMI) 脚本的详细信息，请参阅脚本博客。

语法

relog [<filename> [<filename> ...]] [/a] [/c <path> [<path> ...]] [/cf <filename>] [/f  {bin|csv|tsv|SQL}] [/t <value>] [/o {outputfile|DSN!CounterLog}] [/b <M/D/YYYY> [[<HH>:] <MM>:] <SS>] [/e <M/D/YYYY> [[<HH>:] <MM>:] <SS>] [/config {<filename>|i}] [/q]

参数

参数 说明
filename [filename ...] 指定现有性能计数器日志的路径名称。 可以指定多个输入文件。
-a 追加输出文件，而不是覆盖。 此选项不适用于默认始终追加的SQL格式。
-c path [path ...] 指定要记录的性能计数器路径。 若要指定多个计数器路径，请使用空格分隔这些计数器路径，并将计数器路径括在引号 (中，例如 "path1 path2") 。
-cf 文件名 指定文本文件的路径名称，其中列出了要包含在重新日志文件中的性能计数器。 使用此选项可列出输入文件中的计数器路径，每行一行。 默认设置是重新记录原始日志文件中的所有计数器。
-f {bin | csv | tsv | SQL} 指定输出文件格式的路径名称。 默认格式为 bin。 对于SQL数据库，输出文件指定 DSN!CounterLog. 可以使用 ODBC 管理器来配置 DSN (数据库系统名称) 来指定数据库位置。
-t 值 指定 n 条记录中的示例间隔。 在重新记录文件中包括第 n 个数据点。 默认值为每个数据点。
-o {Outputfile | SQL:DSN!Counter_Log} 指定要在其中写入计数器的输出文件或SQL数据库的路径名称。

注意： 对于 64 位和 32 位版本的 relog.exe，必须在系统上分别在 ODBC 数据源中定义 DSN (64 位和 32 位) 。 使用“SQL Server”ODBC 驱动程序定义 DSN。
-b <M/D/YYYY> [[<HH>:]<MM>:]<SS>] 指定从输入文件复制第一条记录的开始时间。 日期和时间必须采用此确切格式的 M/D/YYYYHH：MM：SS。
-e <M/D/YYYY> [[<HH>:]<MM>:]<SS>] 指定从输入文件复制最后一条记录的结束时间。 日期和时间必须采用此确切格式的 M/D/YYYYHH：MM：SS。
-config {filename | i} 指定包含命令行参数的设置文件的路径名称。 如果使用配置文件，则可以将 -i 用作可在命令行上放置的输入文件列表的占位符。 如果使用命令行，请不要使用 -i。 还可以使用通配符，例如 *.blg 一次性指定多个输入文件名。
-q 显示输入文件中指定的日志文件的性能计数器和时间范围。
-y 通过对所有问题回答“是”来绕过提示。
/? 在命令提示符下显示帮助。

注解

  • 计数器路径的常规格式如下所示： [\<computer>] \<object>[<parent>\<instance#index>] \<counter>] 格式的父组件、实例、索引和计数器组件可能包含有效名称或通配符。 所有计数器都不需要计算机、父组件、实例和索引组件。

  • 根据计数器本身确定要使用的计数器路径。 例如， LogicalDisk 对象具有一个实例 <index>，因此必须提供 <#index> 或通配符。 因此，可以使用以下格式： \LogicalDisk(*/*#*)\\*

  • 相比之下， Process 对象不需要实例 <index>。 因此，可以使用以下格式： \Process(*)\ID Process

  • 如果在 Parent 名称中指定了通配符，则会返回与指定实例和计数器字段匹配的指定对象的所有实例。

  • 如果在 实例 名称中指定了通配符，则当与指定索引对应的所有实例名称与通配符匹配时，将返回指定对象和父对象的所有实例名称。

  • 如果在 Counter 名称中指定了通配符，则返回指定对象的所有计数器。

  • 部分计数器路径字符串与 (匹配，例如，不支持 pro*) 。

  • 计数器文件是列出现有日志中一个或多个性能计数器的文本文件。 以格式复制日志或 /q 输出 <computer>\<object>\<instance>\<counter> 的完整计数器名称。 列出每行的一个计数器路径。

  • 运行时， relog 命令会从输入文件中的每条记录复制指定的计数器，并在必要时转换格式。 计数器文件中允许通配符路径。

  • 使用 /t 参数指定输入文件按每个 nth 记录的间隔插入到输出文件中。 默认情况下，数据会从每个记录重新记录。

  • 可以指定输出日志包括开始时间前 (的记录，即 /b) ，以提供需要格式化值的计算值的计数器的数据。 输出文件将包含输入文件中的最后一条记录，时间戳小于 /e (，即结束时间) 参数。

  • 与 /config 选项一起使用的设置文件的内容应采用以下格式： <commandoption>\<value>其中 <commandoption> 是命令行选项并 <value> 指定其值。

##Q# 示例

若要按 30 个固定间隔重新采样现有跟踪日志，请列出计数器路径、输出文件和格式，键入：

relog c:\perflogs\daily_trace_log.blg /cf counter_file.txt /o c:\perflogs\reduced_log.csv /t 30 /f csv

若要按固定间隔重新采样现有跟踪日志，请键入：

relog c:\perflogs\daily_trace_log.blg /cf counter_file.txt /o c:\perflogs\reduced_log.blg /t 30

若要将现有跟踪日志重新采样到数据库中，请键入：

relog "c:\perflogs\daily_trace_log.blg" -f sql -o "SQL:sql2016x64odbc!counter_log"
