路由

  • 项目
  • 10/08/2022

本文内容

适用范围：Windows Server 2022、Windows Server 2019、Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

显示和修改本地 IP 路由表中的条目。 如果在不使用参数的情况下使用， 路由 会在命令提示符处显示帮助。

重要

仅当 Internet 协议 (TCP/IP) 协议作为组件安装在网络连接中的网络适配器属性中时，此命令才可用。

语法

route [/f] [/p] [<command> [<destination>] [mask <netmask>] [<gateway>] [metric <metric>]] [if <interface>]]

参数

参数 说明
/f 清除未承载路由的所有条目的路由表， (净掩码为 255.255.255.255) 的路由表， 环回网络路由 (目标为 127.0.0.0 且净掩码为 255.0.0.0) ，或者多播路由 (目标为 224.0.0.0 的路由，净掩码为 240.0.0.0) 。 如果此操作与其中一个命令结合使用 (，例如添加、更改或删除) ，则会在运行命令之前清除该表。
/p 与 add 命令一起使用时，指定的路由将添加到注册表中，并在 TCP/IP 协议启动时用于初始化 IP 路由表。 默认情况下，在启动 TCP/IP 协议时，不会保留添加的路由。 与打印命令一起使用时，将显示永久性路由列表。 对于所有其他命令，此参数将被忽略。 永久性路由存储在注册表位置 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\PersistentRoutes。
<command> 指定要运行的命令。 有效命令包括：
  • add - 添加路由。
  • change - 修改现有路由。
  • delete： - 删除路由或路由。
  • print - 打印路由或路由。
<destination> 指定路由的网络目标。 目标可以是 IP 网络地址 (，其中网络地址的主机位设置为 0) 、主机路由的 IP 地址，或默认路由的 0.0.0.0。
<mask> 指定“netmask”值的下一个参数。
<netmask> 指定网络目标子网掩码。 如果未指定，则默认为 255.255.255.255。
<gateway> 指定可访问网络目标和子网掩码定义的地址集的转发或下一跃点 IP 地址。 对于本地附加的子网路由，网关地址是分配给附加到子网的接口的 IP 地址。 对于跨一个或多个路由器提供的远程路由，网关地址是直接可访问的 IP 地址，分配给相邻路由器。
度量 <metric> 指定路由的整数成本度量值（从 1 到 9999），用于在路由表中的多个路由(这些路由最接近匹配要转发的数据包目标地址）间进行选择。 将选择最低度量值的路由。 度量值可以反映跃点数目、路径速度、路径可靠性、路径吞吐量或管理属性。
如果 <interface> 指定访问目标所使用的接口的接口索引。 有关接口及其对应的接口索引的列表，请使用路由打印命令的显示功能。 可以对接口索引使用十进制或十六进制值。 十六进制数以 0x 开头。 省略 if 参数时，接口将通过网关地址确定。
/? 在命令提示符下显示帮助。

注解

  • 路由表 指标 列中的较大值是允许 TCP/IP 根据每个 LAN 接口的 IP 地址、子网掩码和默认网关的配置，自动确定路由表中路由的指标。 默认情况下启用的接口指标自动确定，确定每个接口的速度，并调整每个接口的路由指标，以便最快的接口创建具有最低指标的路由。 若要删除大型指标，请为每个 LAN 连接禁用从 TCP/IP 协议的高级属性中自动确定接口指标。

  • 如果文件夹中存储systemroot\System32\Drivers\\的本地 Networks 文件中存在适当的条目，则可以将名称用于目标。 只要可以通过标准主机名解析技术（例如域名系统 (DNS) 查询、使用存储在systemroot\system32\drivers\\文件夹中的本地 Hosts 文件以及 NetBIOS 名称解析）将名称解析为 IP 地址，就可以将其用于网关。

  • 如果命令是打印或删除的，则可以省略网关参数，通配符可用于目标和网关。 目标值可以是星号(*)指定的通配符值。 如果指定的目标包含星号 (*) 或问号 (？) ，则它被视为通配符，并且仅打印或删除匹配的目标路由。 星号与任何字符串匹配，问号与任何单个字符匹配。 例如，10.\*.1, 192.168.\*127.\*星\*224\*号通配符的所有有效用法。

  • 使用目标掩码和子网掩码的不受支持的组合 (netmask) 值将显示“路由：网关地址错误的 netmask”错误消息。 当目标包含一个或多个位设置为 1 的位位置（其中相应的子网掩码位设置为 0）时，将显示此错误消息。 若要测试此条件，请使用二进制表示法表达目标和子网掩码。 二进制表示法中的子网掩码由一系列 1 位组成，表示目标的网络地址部分，以及一系列 0 位，表示目标的主机地址部分。 检查以确定目标中是否有位设置为 1，该部分是子网掩码定义的主机地址 () 。

示例

若要显示 IP 路由表的整个内容，请键入：

route print

若要在以 10 开头的 IP 路由表中显示路由，请键入：

route print 10.*

若要添加默认网关地址为 192.168.12.1 的默认路由，请键入：

route add 0.0.0.0 mask 0.0.0.0 192.168.12.1

若要将路由添加到目标 10.41.0.0，子网掩码为 255.255.0.0，下一跃点地址为 10.27.0.1，请键入：

route add 10.41.0.0 mask 255.255.0.0 10.27.0.1

若要将永久性路由添加到目标 10.41.0.0，子网掩码为 255.255.0.0，以及 10.27.0.1 的下一跃点地址，请键入：

route /p add 10.41.0.0 mask 255.255.0.0 10.27.0.1

若要将路由添加到目标 10.41.0.0，子网掩码为 255.255.0.0，下一跃点地址为 10.27.0.1，以及成本指标为 7，请键入：

route add 10.41.0.0 mask 255.255.0.0 10.27.0.1 metric 7

若要将路由添加到目标 10.41.0.0，子网掩码为 255.255.0.0，下一跃点地址为 10.27.0.1，并使用接口索引0x3，键入：

route add 10.41.0.0 mask 255.255.0.0 10.27.0.1 if 0x3

若要删除路由到目标 10.41.0.0，子网掩码为 255.255.0.0，请键入：

route delete 10.41.0.0 mask 255.255.0.0

若要删除以 10 开头的 IP 路由表中的所有路由，请键入：

route delete 10.*

若要将目标为 10.41.0.0 的路由的下一跃点地址和子网掩码 255.255.0.0.0 从 10.27.0.1 更改为 10.27.0.25，请键入：

route change 10.41.0.0 mask 255.255.0.0 10.27.0.25
