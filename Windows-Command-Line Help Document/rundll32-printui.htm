rundll32 printui.dll，PrintUIEntry

  • 项目
  • 09/21/2022

本文内容

适用范围：Windows Server 2022、Windows Server 2019、Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

自动执行许多打印机配置任务。 printui.dll是包含打印机配置对话框使用的函数的可执行文件。 还可以从脚本或命令行批处理文件中调用这些函数，也可以从命令提示符以交互方式运行这些函数。

语法

rundll32 printui.dll PrintUIEntry [baseparameter] [modificationparameter1] [modificationparameter2] [modificationparameterN]

还可以使用以下备用语法，尽管本主题中的示例使用前面的语法：

rundll32 printui.dll,PrintUIEntry [baseparameter] [modificationparameter1] [modificationparameter2] [ModificationParameterN]

rundll32 printui PrintUIEntry [baseparameter] [modificationparameter1] [modificationparameter2] [modificationparameterN]

rundll32 printui,PrintUIEntry [baseparameter] [modificationparameter1] [modificationparameter2] [modificationparameterN]

参数

有两种类型的参数：基参数和修改参数。 基参数指定命令要执行的函数。 只有其中一个参数可以出现在给定的命令行中。 然后，如果基参数适用于基参数，则可以使用一个或多个修改参数来修改基参数， (并非所有修改参数都受所有基参数) 支持。

基本参数 说明
/dl 删除本地打印机。
/dn 删除网络打印机连接。
/dd 删除打印机驱动程序。
/e 显示给定打印机的打印首选项。
/ga 添加每台计算机打印机连接， (该计算机上的任意用户在登录) 时可以使用该连接。
/ge 显示计算机上的每台计算机打印机连接。
/gd 下次用户登录) 时删除每个计算机打印机连接， (删除该连接。
/ia 使用 .inf 文件安装打印机驱动程序。
/id 使用添加打印机驱动程序向导安装打印机驱动程序。
/if 使用 .inf 文件安装打印机。
/ii 通过添加打印机向导和 .inf 文件安装打印机。
/il 使用添加打印机向导安装打印机。
/in 连接到远程网络打印机。
/ip 使用网络打印机安装向导安装打印机， (打印管理) 的用户界面提供。
/k 在打印机上打印测试页。
/o 显示打印机的队列。
/p 显示打印机的属性。 使用此参数时，还必须为修改参数 /n[name] 指定值。
/s 显示打印服务器的属性。 如果要查看本地打印服务器，则无需使用修改参数。 但是，如果要查看远程打印服务器，则必须指定 /c[name] 修改参数。
/Ss 指定将存储打印机的信息类型。 如果未指定 /S 的值 ，则默认行为就像指定了所有值一样。 将此基参数与放置在命令行末尾的以下值一起使用：
  • 2：存储打印机printER_INFO_2结构中包含的信息。 此结构包含有关打印机的基本信息，例如其名称、服务器名称、端口名称和共享名称。
  • 7：用于存储printER_INFO_7结构中包含的目录服务信息。
  • c：存储打印机的颜色配置文件信息。
  • d：存储打印机特定数据，例如打印机的硬件 ID。
  • s：存储打印机的安全描述符。
  • g：将信息存储在打印机的全局 DEVmode 结构中。
  • m：存储打印机的最小设置。 这相当于指定 2d 和 g。
  • u：将信息存储在每用户 DEVmode 结构的打印机中。
/Sr 指定还原打印机的相关信息以及如何处理设置中的冲突。 与命令行末尾放置的以下值一起使用：
  • 2：还原打印机printER_INFO_2结构中包含的信息。 此结构包含有关打印机的基本信息，例如其名称、服务器名称、端口名称和共享名称。
  • 7：还原printER_INFO_7结构中包含的目录服务信息。
  • c：还原打印机的颜色配置文件信息。
  • d：还原打印机特定数据，例如打印机的硬件 ID。
  • s：还原打印机的安全描述符。
  • g：还原打印机全局 DEVmode 结构中的信息。
  • m：还原打印机的最小设置。 这等效于指定 2、 d 和 g。
  • u 还原每用户 DEVmode 结构打印中的信息。
  • r：如果存储在文件中的打印机名称与要还原到的打印机的名称不同，请使用当前打印机名称。 不能使用 f 指定此项。 如果未指定 r 和 f ，并且名称不匹配，则还原设置将失败。
  • f：如果存储在文件中的打印机名称与要还原到的打印机的名称不同，请使用文件中的打印机名称。 不能使用 r 指定此项。 如果未指定 f 和 r ，并且名称不匹配，则还原设置会失败。
  • p：如果从中还原的文件中的端口名称与要还原到的打印机的当前端口名称不匹配，则使用打印机的当前端口名称。
  • h：如果还原到的打印机无法使用保存的设置文件中的资源共享名称共享，则尝试使用当前共享名称或新的生成的共享名称共享打印机（如果未指定 H 或 h ，并且要还原为无法与保存共享名称共享的打印机） 然后还原失败。
  • h：如果还原到无法与保存的共享名称共享打印机，则不要共享打印机。 如果未指定 H 和 h ，并且要还原的打印机无法与保存的共享名称共享，则还原将失败。
  • i：如果保存的设置文件中的驱动程序与要还原到的打印机的驱动程序不匹配，则还原会失败。
/Xg 检索打印机的设置。
/Xs 设置打印机的设置。
/y 将要安装的打印机设置为默认打印机。
/? 显示命令及其关联参数的产品内帮助。
@[file] 指定命令行参数文件，并将该文件中的文本直接插入命令行。
修改参数 说明
/a[file] 指定二进制文件名。
/b[name] 指定基本打印机名称。
/c[name] 如果要执行的操作在远程计算机上，则指定计算机名称。
/f[file] 根据所执行的任务，通用命名约定 (UNC) .inf 文件名或输出文件名的路径和名称。 使用 /F[file] 指定依赖的 .inf 文件。
/F[file] 指定 使用 /f[file] 指定的 .inf 文件的 UNC 路径和名称。
/h[architecture] 指定驱动程序体系结构。 使用以下项之一： x86、 x64 或 Itanium。
/j[provider] 指定打印提供程序名称。
/l[path] 指定正在使用的打印机驱动程序文件的 UNC 路径。
/m[model] 指定驱动程序模型名称。 (可以在 .inf file.) 中指定此值
/n[name] 指定打印机名称。
/q 运行没有向用户发出通知的命令。
/r[port] 指定端口名称。
/U 指定在已安装打印机驱动程序时使用现有打印机驱动程序。
/t[#] 指定要从零开始的索引页。
/v[version] 指定驱动程序版本。 如果未为 /K 指定值，则必须指定以下值之一： 类型 2 - 内核模式 或 类型 3 - 用户模式。
/W 如果在 由 /f 指定的 .inf 文件中找不到驱动程序，则提示用户输入驱动程序。
/Y 指定不应自动生成打印机名称。
/z 指定不自动共享所安装的打印机。
/K 将参数 /h[architecture] 的含义更改为接受 2 代替 x86、 3 代替 x64 或 4 代替 Itanium。 它还将参数 /v[version] 的值更改为在类型 2 中接受 2 - 内核模式，3 代替类型 3 - 用户模式。
/Z 共享正在安装的打印机。 仅与 /if 参数一起使用。
/Mw[message] 在提交命令行中指定的更改之前，向用户显示警告消息。
/Mq[message] 在提交命令行中指定的更改之前，向用户显示确认消息。
/W[flags] 指定添加打印机向导、添加打印机驱动程序向导和网络打印机安装向导的任何参数或选项。

r：允许从最后一页重启向导。
/G[flags] 指定要使用的全局参数和选项。

w：禁止向用户显示安装驱动程序警告。

注解

  • PrintUIEntry 关键字区分大小写，必须输入此命令的语法，并且本主题中的示例中显示的确切大写。

  • 有关更多示例，请在命令提示符类型： rundll32 printui.dll，PrintUIEntry /？

示例

若要为计算机添加新的远程打印机、printer1，客户端 1 对于运行此命令的用户帐户可见，请键入：

rundll32 printui.dll PrintUIEntry /in /n\\client1\printer1

若要使用添加打印机向导和 .inf 文件添加打印机，位于驱动器 c： Infpath 上的 InfFile.inf，请键入：

rundll32 printui.dll PrintUIEntry /ii /f c:\Infpath\InfFile.inf

若要删除计算机上的现有打印机、printer1，请键入：

rundll32 printui.dll PrintUIEntry /dn /n\\client1\printer1

若要为计算机的所有用户添加每台计算机打印机连接，printer2，请在用户登录) 时键入 (该连接：

rundll32 printui.dll PrintUIEntry /ga /n\\client2\printer2

若要删除每台计算机打印机连接，打印机 2（对于计算机的所有用户，Client2）键入 (当用户登录) 时，将删除连接：

rundll32 printui.dll PrintUIEntry /gd /n\\client2\printer2

若要查看打印服务器的属性，printServer1，请键入：

rundll32 printui.dll PrintUIEntry /s /t1 /c\\printserver1

若要查看打印机的属性，请键入：

rundll32 printui.dll PrintUIEntry /p /n\\printer3
