本文内容

在一组文件中为每个文件运行指定的命令。

语法

for {%% | %}<variable> in (<set>) do <command> [<commandlineoptions>]

参数

参数 说明
{%% \| %}<variable> 必需。 表示可替换参数。 使用单一百分号 (%) 在命令提示符处执行 for 命令。 使用双百分号 (%%) 在批处理文件中执行 for 命令。 变量区分大小写，必须使用字母值（如 %a、 %b 或 %c）来表示它们。
(<set>) 必需。 指定要对其运行命令的一个或多个文件、目录或文本字符串，或一系列值。 需要使用括号。
<command> 必需。 指定要对每个文件、目录或文本字符串或 集中包含的值范围执行命令。
<commandlineoptions> 指定要用于指定命令的任何命令行选项。
/? 在命令提示符下显示帮助。

备注

  • 可以在批处理文件中或直接从命令提示符使用此命令。

  • 以下属性适用于 for 命令：

    • 此命令将替换或%% variable替换为% variable指定集中的每个文本字符串，直到指定的命令处理所有文件。

    • 变量名称区分大小写，全局且一次不能超过 52 个。

    • 为避免与批处理参数混淆，%0可以通过此方式%9将除数字 0 到 9 之外的任何字符用于变量。 对于简单的批处理文件，单个字符（如 %%f 将正常工作）。

    • 可以在复杂批处理文件中对 变量 使用多个值来区分不同的可替换变量。

  • set 参数可以表示单个文件组或多个文件组。 可以使用通配符 (* 和 ？) 指定文件集。 下面是有效的文件集：

    (*.doc)
(*.doc *.txt *.me)
(jan*.doc jan*.rpt feb*.doc feb*.rpt)
(ar??1991.* ap??1991.*)

  • 使用此命令时， 集 中的第一个值将替换 % variable 或 %% variable，然后指定的命令处理此值。 这将继续，直到处理与 设置 值相对应的所有文件 (或文件组) 。

  • 中 和 确实 不是参数，但必须将它们与此命令一起使用。 如果省略其中任一关键字，将显示一条错误消息。

  • 如果在默认) (启用命令扩展，则支持以下附加形式的 命令 扩展：

    • 仅目录： 如果 set 包含 (* 或 ？) 的通配符，则指定 命令 将针对每个目录执行 (而不是指定目录) 中与集匹配的文件 集。 语法为：

      for /d {%%|%}<Variable> in (<Set>) do <Command> [<CommandLineOptions>]

    • 递归的： 走根于 drive：path 的目录树，并在树的每个目录中执行 for 语句。 如果在 /r 之后未指定任何目录，则当前目录将用作根目录。 如果 set 只是单个句点 (.) ，则它仅枚举目录树。 语法为：

      for /r [[<drive>:]<path>] {%%|%}<variable> in (<set>) do <command> [<commandlinepptions>]

    • 循环访问一系列值： 使用迭代变量设置起始值 (start#) ，然后单步执行一组值范围，直到该值超出 set 结束值 (end#) 。 /l 将通过比较 start# 与 end# 来执行迭代。 如果 start# 小于 end#，则命令将执行。 当迭代变量超过 end#时，命令 shell 将退出循环。 还可以使用负 步骤# 逐个执行缩小值的范围。 例如， (1，1，5) 生成序列 1 2 3 4 5， (5，-1，1) 生成序列 5 4 3 2 1。 语法为：

      for /l {%%|%}<variable> in (<start#>,<step#>,<end#>) do <command> [<commandlinepptions>]

    • 迭代和文件分析： 使用文件分析处理命令输出、字符串和文件内容。 使用迭代变量定义要检查的内容或字符串，并使用各种 分析keyword 选项进一步修改分析。 使用 parsingkeywords 令牌选项指定应作为迭代变量传递哪些令牌。 在没有标记选项的情况下使用时， /f 将仅检查第一个令牌。

      文件分析包括读取输出、字符串或文件内容，然后将其分解为单个文本行，并将每行分析为零个或多个标记。 然后调用 for 循环，并将迭代变量值设置为令牌。 默认情况下， /f 会从每个文件的每行传递第一个空白分隔令牌。 跳过空白行。

      语法为：

      for /f [<parsingkeywords>] {%%|%}<variable> in (<set>) do <command> [<commandlinepptions>]
for /f [<parsingkeywords>] {%%|%}<variable> in (<literalstring>) do <command> [<commandlinepptions>]
for /f [<parsingkeywords>] {%%|%}<variable> in ('<command>') do <command> [<commandlinepptions>]

      set 参数指定一个或多个文件名。 在移动到 集中的下一个文件之前，将打开、读取和处理每个文件。 若要重写默认分析行为，请指定 parsingkeyword。 这是一个带引号的字符串，其中包含一个或多个关键字来指定不同的分析选项。

      如果使用 usebackq 选项，请使用以下语法之一：

      for /f [usebackq <parsingkeywords>] {%%|%}<variable> in (<Set>) do <command> [<commandlinepptions>]
for /f [usebackq <parsingkeywords>] {%%|%}<variable> in ('<LiteralString>') do <command> [<commandlinepptions>]
for /f [usebackq <parsingkeywords>] {%%|%}<variable> in (`<command>`) do <command> [<commandlinepptions>]

      下表列出了可用于 分析keyword 的分析关键字。

      关键字 说明
      eol=<c> 指定行字符的末尾 (一个字符) 。
      skip=<n> 指定要在文件开头跳过的行数。
      delims=<xxx> 指定分隔符集。 这将替换默认分隔符集的空间和制表符。
      tokens=<x,y,m–n> 指定要将每个行中的标记传递给每个迭代的 for 循环。 因此，会分配其他变量名称。 m-n 指定从 m到 n个标记的范围。 如果 token= 字符串中的最后一个字符是星号 (*) ，则会分配一个附加变量，并在分析的最后一个标记之后接收行上的剩余文本。
      usebackq 指定将后引号字符串作为命令运行，将单引号字符串用作文本字符串，或者对于包含空格的长文件名，允许将文件名括在 <set>双引号中。

    • 变量替换： 下表列出了任何变量 I) 的可选语法 (：

      带修饰符的变量 说明
      %~I %I展开可删除任何周围引号。
      %~fI %I展开到完全限定的路径名称。
      %~dI %I仅扩展到驱动器号。
      %~pI %I仅扩展到路径。
      %~nI %I仅扩展到文件名。
      %~xI %I仅扩展到文件扩展名。
      %~sI 展开路径以仅包含短名称。
      %~aI %I展开到文件的文件属性。
      %~tI %I展开到文件的日期和时间。
      %~zI %I扩展为文件大小。
      %~$PATH:I 搜索 PATH 环境变量中列出的目录，并展开 %I 到找到的第一个目录的完全限定名称。 如果未定义环境变量名称，或者搜索找不到文件，则此修饰符将展开为空字符串。

      下表列出了可用于获取复合结果的修饰符组合。

      具有组合修饰符的变量 说明
      %~dpI %I仅扩展到驱动器号和路径。
      %~nxI %I仅扩展到文件名和扩展名。
      %~fsI %I展开为仅包含短名称的完整路径名称。
      %~dp$PATH:I 搜索 PATH 环境变量 %I 中列出的目录，并展开到找到的第一个驱动器号和路径。
      %~ftzaI %I展开到类似于 dir 的输出行。

      在上面的示例中，可以将 PATH 替换为 %I 其他有效值。 变量名称的有效结束%~语法。

      通过使用大写变量名称（例如 %I），可以使代码更易于阅读，并避免与修饰符混淆，这些修饰符不区分大小写。

  • 分析字符串：可以通过包装<literalstring>在一个即时字符串上使用分析逻辑：双引号 (，而不使用 for /f usebackq) ，或在具有 usebackq) 的单引号 (中使用 --例如， (MyString) 或 ('MyString') 。 <literalstring> 被视为文件中的单个输入行。 在双引号中分析 <literalstring> 时，命令符号（如 (\ & | > < ^) ）被视为普通字符。

  • 分析输出： 可以使用 for /f 该命令来分析命令的输出，方法是在括号之间放置后引号 <command> 。 它被视为命令行，该命令行传递给子Cmd.exe。 输出将捕获到内存中，并像分析文件一样进行分析。

示例

若要在批处理文件中 使用 ，请使用以下语法：

for {%%|%}<variable> in (<set>) do <command> [<commandlineoptions>]

若要使用可替换变量 %f 显示当前目录中扩展名为.doc或.txt的所有文件的内容，请键入：

for %f in (*.doc *.txt) do type %f

在前面的示例中，当前目录中具有.doc或.txt扩展名的每个文件都替换为 %f 变量，直到显示每个文件的内容。 若要在批处理文件中使用此命令，请将 %f 的每个匹配项替换为 %%f。 否则，将忽略变量并显示错误消息。

若要分析文件，请忽略注释行，键入：

for /f eol=; tokens=2,3* delims=, %i in (myfile.txt) do @echo %i %j %k

此命令分析 myfile.txt中的每个行。 它忽略以分号开头的行，并将第二和第三个标记从每行 传递到正文 ( 标记用逗号或空格分隔) 。 for 语句的正文引用 %i 以获取第二个令牌，%j 获取第三个令牌，%k 获取其余所有令牌。 如果提供的文件名包含空格，请使用文本 (的引号，例如文件名) 。 若要使用引号，必须使用 usebackq。 否则，引号将解释为定义要分析的文本字符串。

%i 在 for 语句中显式声明。 %j 和 %k 是使用 token=隐式声明的。 可以使用 token= 指定最多 26 个令牌，前提是它不会导致尝试声明高于字母 z 或 Z 的变量。

若要通过在括号之间放置 集 来分析命令的输出，请键入：

for /f "usebackq delims==" %i in (`set`) do @echo %i
