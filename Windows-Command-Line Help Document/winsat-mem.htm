winsat mem

  • 项目
  • 09/21/2022

本文内容

像在多媒体处理中使用的一样，以将大型内存反射到内存缓冲区副本的方式测试系统内存带宽。

语法

winsat mem <parameters>

参数

参数 说明
-up 使用一个线程强制进行内存测试。 默认值是每个物理 CPU 或核心运行一个线程。
-rn 指定评估的线程应按正常优先级运行。 默认以优先级 15 运行。
-nc 指定评估应分配内存，并标记为"未缓存"。 这意味着将绕过处理器的缓存进行复制操作。 默认在缓存空间中运行。
-do <n> 指定源缓冲区末尾和目标缓冲区开头之间的距离（以字节为单位）。 默认值为 64 字节。 允许的最大目标偏移量为 16MB。 指定无效的目标偏移量将导致错误。
注意：零是 n 的有效
但负数不是。
-将 <n> 指定评估的最小运行时间（以秒表示）。 默认值为 2.0。 最小值为 1.0。 最大值为 30.0。
注意：当结合使用
，指定大于 -maxt 值的 - mint 值将导致错误。
-maxt <n> 指定评估的最大运行时间（以秒表示）。 默认值为 5.0。 最小值为 1.0。 最大值为 30.0。 如果与 -mint 参数结合使用，评估将在 -mint 中指定的时间段后开始定期对结果进行 统计检查。 如果统计检查通过，则评估将在 -maxt 中指定的时间段之前完成。 如果评估在 -maxt 中指定的时段内运行，而不满足统计检查，则评估将在该时间完成并返回它收集的结果。
-buffersize <n> 指定内存复制测试应该使用的缓冲区大小。 此量将按 CPU 分配两次，这决定了从一个缓冲区复制到另一个缓冲区的数据量。 默认值为 16MB。 此值舍入为最接近的 4 KB 边界。 最大值为 32MB。 最小值为 4 KB。 指定无效的缓冲区大小将导致错误。
-v 将详细输出发送到 STDOUT，包括状态和进度信息。 任何错误也将写入命令窗口。
-xml <文件名> 将评估的输出另存为指定的 XML 文件。 如果指定的文件存在，它将被覆盖。
-idiskinfo 将物理卷和逻辑磁盘的信息保存为 XML 输出中 SystemConfig> 部分的一部分。
-iguid 在 XML 输出文件中 (GUID) 全局唯一标识符。
-note note text 将注释文本添加到 >XML 输出文件的 note 节。
-icn 在 XML 输出文件中包括本地计算机名称。
-eef 枚举 XML 输出文件中额外的系统信息。

示例

  • 若要运行评估至少 4 秒且不超过 12 秒，使用 32MB 缓冲区大小，将结果以 XML 格式 保存到文件memtest.xml。 
    winsat mem -mint 4.0 -maxt 12.0 -buffersize 32MB -xml memtest.xml

注解

  • 本地管理员组或等效的成员身份是使用 winsat 所需的最低要求。 必须从提升的命令提示符窗口执行该命令。
