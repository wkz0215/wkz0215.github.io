defrag

  • 项目
  • 09/21/2022

本文内容

适用于：Windows Server 2022、Windows Server 2019、Windows 10、Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

在本地卷上查找和合并碎片文件以提高系统性能。

运行此命令所需的最低要求是本地 Administrators 组或等效的成员身份。

语法

defrag <volumes> | /c | /e <volumes>    [/h] [/m [n]| [/u] [v]]
defrag <volumes> | /c | /e <volumes> /a [/h] [/m [n]| [/u] [v]]
defrag <volumes> | /c | /e <volumes> /x [/h] [/m [n]| [/u] [v]]
defrag <volume> [<parameters>]

参数

参数 说明
<volume> 指定要碎片整理或分析的卷的驱动器号或装入点路径。
/a 对指定的卷执行分析。
/c 在所有卷上执行此操作。
/d 执行传统的碎片整理 (这是默认) 。 但是，在分层卷上，传统碎片整理仅在容量层上执行。
/e 对除指定卷以外的所有卷执行该操作。
/g 优化指定卷上的存储层。
/h 如果默认为低优先级， (以正常优先级) 。
/i [n] 层优化在每个卷上最多运行 n 秒。
/k 对指定的卷执行板合并。
/l 对指定的卷执行重试。
/m [n] 在后台并行运行每个卷上的操作。 最多 n 个线程并行优化存储层。
/o 针对每种媒体类型执行适当的优化。
/t 跟踪已在指定卷上执行的操作。
/U 在屏幕上打印操作的进度。
/v 打印包含碎片统计信息的详细输出。
/x 对指定的卷执行可用空间合并。
/? 显示此帮助信息。

注解

  • 无法对特定的文件系统卷或驱动器进行碎片整理，包括：

    • 文件系统锁定的卷。

    • 将文件系统标记为脏的卷，指示可能的损坏。
      必须先运行 chkdsk ，然后才能对此卷或驱动器进行碎片整理。 可以使用 命令确定卷是否脏 fsutil dirty 。

    • 网络驱动器。

    • CD-ROM。

    • 不是 NTFS、 ReFS、 Fat 或 Fat32 的文件系统卷。

  • 若要执行该过程，你必须是本地计算机上 Administrators 组的成员，或你必须已被委派适当的权限。 如果计算机已加入域，则 Domain Admins 组的成员也许能够执行该过程。 作为最佳安全做法，请考虑使用 运行方式 执行此过程。

  • 卷必须至少具有 15% 的可用空间，碎片 整理 可以完全、充分地进行碎片整理。 碎片整理 使用此空间作为文件片段的排序区域。 如果卷的可用空间小于 15%，碎片 整理 将仅对卷进行部分碎片整理。 若要增加卷上的可用空间，请删除不需要的文件或将它们移到另一个磁盘。

  • 当 碎片整理 正在分析和碎片整理卷时，它会显示闪烁的光标。 完成 对 卷的分析和碎片整理后，它将显示分析报告、碎片整理报告或两个报表，然后退出命令提示符。

  • 默认情况下 ，如果 未指定 /a 或 /v 参数，碎片整理将显示分析和碎片整理报表的摘要。

  • 可以通过键入FileName.txt> 将报表发送到文本文件，其中 FileName.txt是指定的文件名。> 例如：defrag volume /v > FileName.txt

  • 若要中断碎片整理过程，在命令行上按 CTRL+C。

  • 运行 defrag 命令和磁盘碎片整理程序是互斥的。 如果使用磁盘碎片整理器对卷进行碎片整理，并且在命令行中运行 defrag 命令， 则 defrag 命令将失败。 相反，如果运行 defrag 命令并打开磁盘碎片整理程序，则磁盘碎片整理器中的碎片整理选项将不可用。

示例

若要在提供进度和详细输出的同时对驱动器 C 上的卷进行碎片整理，请键入：

defrag c: /u /v

若要在后台对驱动器 C 和 D 上的卷进行碎片整理，请键入：

defrag c: d: /m

若要对安装在驱动器 C 上的卷执行碎片分析并提供进度，请键入：

defrag c: mountpoint /a /u

若要对具有正常优先级的所有卷进行碎片整理并提供详细输出，请键入：

defrag /c /h /v

计划任务

碎片整理过程将计划任务作为维护任务运行，通常每周运行一次。 作为管理员，可以使用"优化驱动器"应用更改 任务的运行 方式。

  • 从计划任务运行时， 碎片整理 对 SSD 使用以下策略准则：

    • 传统优化过程。 包括 传统的碎片整理，例如移动文件使其相当连续和 重试。 此操作每月执行一次。 但是， 如果同时跳过 传统的碎片整理和 重试 ，则 不 运行分析。 更改计划任务的频率不会影响每月一次 SSD 的频率。

    • 如果在 SSD 上手动运行传统的碎片整理，则在正常计划运行之间，下一个计划的任务运行将执行分析和重试，但会跳过该 SSD 上的传统碎片整理。

    • 如果跳过分析，在"优化驱动器"应用中不会看到更新的"上次运行时间"。 因此，" 上次运行时间" 可能最多为一个月。

    • 你可能会发现，计划任务未对所有卷进行碎片整理。 这通常是因为：

      • 进程不会唤醒计算机以运行。

      • 计算机未接通电源。 如果计算机以电池电源运行，则进程不会运行。

      • 计算机已启动备份 (从空闲状态) 。
