winnt32

  • 项目
  • 09/21/2022

本文内容

适用范围：Windows Server 2022、Windows Server 2019、Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

在 Windows Server 2003 中安装或升级到产品。 可以在运行 Windows 95、Windows 98、Windows 千年版、Windows NT、Windows 2000、Windows XP 或 Windows Server 2003 中的产品的计算机的命令提示符处运行 winnt32。 如果在运行 Windows NT 版本 4.0 的计算机上运行 winnt32，则必须首先应用 Service Pack 5 或更高版本。

语法

winnt32 [/checkupgradeonly] [/cmd: <CommandLine>] [/cmdcons] [/copydir:{i386|ia64}\<FolderName>] [/copysource: <FolderName>] [/debug[<Level>]:[ <FileName>]] [/dudisable] [/duprepare: <pathName>] [/dushare: <pathName>] [/emsport:{com1|com2|usebiossettings|off}] [/emsbaudrate: <BaudRate>] [/m: <FolderName>]  [/makelocalsource] [/noreboot] [/s: <Sourcepath>] [/syspart: <DriveLetter>] [/tempdrive: <DriveLetter>] [/udf: <ID>[,<UDB_File>]] [/unattend[<Num>]:[ <AnswerFile>]]

参数

参数 说明
/checkupgradeonly 检查计算机是否与 Windows Server 2003 中的产品升级兼容性。

如果将此选项用于 /unattend，则无需用户输入。 否则，结果将显示在屏幕上，可以在指定的文件名下保存它们。 默认文件名在 systemroot 文件夹中 upgrade.txt 。
/cmd 指示安装程序在安装的最后阶段之前执行特定命令。 在计算机重启后以及安装程序收集必要的配置信息之后，但在安装完成之前，会发生此情况。
<CommandLine> 指定要在设置的最后阶段之前执行的命令行。
/cmdcons 在基于 x86 的计算机上，将恢复控制台安装为启动选项。 恢复控制台是一个命令行界面，可从中执行启动和停止服务和访问本地驱动器 (包括格式化为 NTFS) 的驱动器等任务。 设置完成后，只能使用 /cmdcons 选项。
/copydir 在安装操作系统文件的文件夹中创建一个附加文件夹。 例如，对于基于 x86 和 x64 的计算机，可以在 i386 源文件夹中创建名为 Private_drivers 的文件夹进行安装，并将驱动程序文件放在文件夹中。 键入 /copydir：i386\Private_drivers将该文件夹安装程序复制到新安装的计算机，使新文件夹位置 systemroot\Private_drivers。

- i386 指定 i386
- ia64 指定 ia64

可以使用 /copydir 根据需要创建尽可能多的其他文件夹。
<FolderName> 指定为保存网站修改而创建的文件夹。
/copysource 在安装操作系统文件的文件夹内创建一个临时附加文件夹。 可以使用 /copysource 根据需要创建任意数量的其他文件夹。

与 /copydir 创建的文件夹不同，安装程序完成后会删除 /copysource 文件夹。
/debug 在指定的级别创建调试日志，例如 /debug4：Debug.log。 默认日志文件为 C：\systemroot\winnt32.log 和
<level> 级别值和说明

- 0：严重错误
- 1：错误
- 2：默认级别。 警告
- 3：信息
- 4：调试的详细信息

每个级别都包含其下面的级别。
/dudisable 防止动态更新运行。 如果没有动态更新，安装程序仅使用原始安装程序文件运行。 即使使用应答文件并在该文件中指定动态更新选项，此选项也会禁用动态更新。
/duprepare 在安装共享上执行准备，以便它可以与从Windows 更新网站下载的动态更新文件一起使用。 然后，此共享可用于为多个客户端安装 Windows XP。
<路径> 指定完整路径名称。
/dushare 指定以前下载的动态更新文件 (更新的文件的共享，以便与Windows 更新网站中的安装程序) 一起使用，以及之前运行 /duprepare：< pathName> 的共享。 在客户端上运行时，指定客户端安装将使用 pathName> 中指定的共享上的<更新文件。
/emsport 在安装期间以及安装服务器操作系统后启用或禁用紧急管理服务。 使用紧急管理服务，可以在紧急情况下远程管理服务器，这些服务器通常需要本地键盘、鼠标和监视器，例如网络不可用或服务器无法正常工作。 紧急管理服务具有特定的硬件要求，仅适用于 Windows Server 2003 中的产品。

- com1 仅适用于基于 x86 的计算机， (不适用于基于 Itanium 体系结构的计算机) 。
- com2is 仅适用于基于 x86 的计算机， (不适用于基于 Itanium 体系结构的计算机) 。
- 默认值。 使用 BIOS 串行端口控制台重定向中指定的设置， (SPCR) 表，或通过 EFI 控制台设备路径在基于 Itanium 体系结构的系统中使用。 如果指定 usebiossettings ，并且没有 SPCR 表或适当的 EFI 控制台设备路径，则不会启用紧急管理 Serices。
- off 禁用紧急管理服务。 稍后可以通过修改启动设置来启用它。
/emsbaudrate 对于基于 x86 的计算机，指定紧急管理服务的波特率。 (此选项不适用于基于 Itanium 体系结构的计算机。) 必须与 /emsport：com1 或 /emsport：com2 (一起使用，否则将忽略 /emsbaudrate) 。
<BaudRate> 指定 9600、19200、57600 或 115200 的波特率。 9600 是默认值。
/m 指定设置从备用位置复制替换文件。 指示安装程序先在备用位置查找，如果存在文件，请使用它们，而不是默认位置中的文件。
/makelocalsource 指示安装程序将所有安装源文件复制到本地硬盘。 在从 cd 安装时使用 /makelocalsource ，在安装中稍后不可用时提供安装文件。
/noreboot 指示安装程序在安装文件复制阶段完成后不重启计算机，以便可以运行另一个命令。
/s 指定安装文件的源位置。 若要同时从多个服务器复制文件，请多次键入 /s：<Sourcepath> 选项， (最多八个) 。 如果多次键入该选项，则指定的第一台服务器必须可用，否则安装程序将失败。
<Sourcepath> 指定完整的源路径名称。
/syspart 在基于 x86 的计算机上，指定可以将安装程序启动文件复制到硬盘，将磁盘标记为活动，然后将磁盘安装到另一台计算机。 启动该计算机时，会自动启动下一阶段的设置。

必须始终将 /tempdrive 参数与 /syspart 参数一起使用。

可以在运行 Windows NT 4.0、Windows 2000、Windows XP 或 Windows Server 2003 中的产品的基于 x86 的计算机上使用 /syspart 选项启动 winnt32。 如果计算机运行Windows NT版本 4.0，则需要 Service Pack 5 或更高版本。 计算机无法运行 Windows 95、Windows 98 或 Windows 千年版。
<DriveLetter> 指定驱动器号。
/tempdrive 指示安装程序将临时文件放置在指定的分区上。

对于新的安装，服务器操作系统也将安装在指定的分区上。

对于升级， /tempdrive 选项仅影响临时文件的位置;操作系统将在运行 winnt32 的分区中升级。
/udf 指示安装程序用来指定唯一性数据库 (UDB) 文件如何修改应答文件的标识符 (<ID>) ， (查看 /unattend 选项) 。 UDB 替代应答文件中的值，标识符确定使用 UDB 文件中的值。 例如， /udf：RAS_user，Our_company.udb 替代为 Our_company.udb 文件中RAS_user标识符指定的设置。 如果未 <指定UDB_file> ，安装程序会提示用户插入包含 $Unique$.udb 文件的磁盘。
<ID> 指示用于指定唯一性数据库 (UDB) 文件如何修改应答文件的标识符。
<UDB_file> 指定唯一性数据库 (UDB) 文件。
/unattend 在基于 x86 的计算机上，使用 Service Pack 5 或更高版本的 Service Pack 5) 或更高版本升级以前版本的 Windows NT 4.0 Server (，或者在无人参与安装模式下Windows 2000。 所有用户设置都是从以前的安装中获取的，因此在安装过程中不需要用户干预。
<num> 指定安装程序完成复制文件的时间和重启计算机之间的秒数。 可以在运行 Windows 98、Windows Millennium edition、Windows NT、Windows 2000、Windows XP 或 Windows Server 2003 中的产品的任何计算机上使用 <Num>。 如果计算机运行Windows NT版本 4.0，则需要 Service Pack 5 或更高版本。
<AnswerFile> 提供自定义规范的设置
/? 在命令提示符下显示帮助。

注解

如果要在客户端计算机上部署 Windows XP，则可以使用 Windows XP 附带的winnt32.exe版本。 部署Windows XP 的另一种方法是使用 winnt32.msi，该Windows安装程序是 IntelliMirror 技术集的一部分。 有关客户端部署的详细信息，请参阅 Windows Server 2003 部署工具包，如使用 Windows 部署和资源工具包中所述。

在基于 Itanium 的计算机上，winnt32 可以从可扩展固件接口 (EFI) 或 Windows Server 2003 Enterprise、Windows Server 2003 R2 Enterprise、Windows Server 2003 R2 Datacenter 或 Windows Server 2003 Datacenter 运行。 此外，在基于 Itanium 体系结构的计算机上， /cmdcons 和 /syspart 不可用，与升级相关的选项不可用。 有关硬件兼容性的详细信息，请参阅 硬件兼容性。 有关使用动态更新和安装多个客户端的详细信息，请参阅 Windows Server 2003 部署工具包，如使用 Windows 部署和资源工具包中所述。 有关修改启动设置的信息，请参阅 Windows Server 2003 的Windows部署和资源工具包。 有关详细信息，请参阅使用Windows部署和资源工具包。 使用 /unattend 命令行选项自动设置确认已阅读并接受 Windows Server 2003 的 Microsoft 许可协议。 在使用此命令行选项代表你自己的组织安装 Windows Server 2003 之前，必须确认最终用户 (个人或单个实体) 是否已收到、读取并接受该产品的 Microsoft 许可协议条款。 OEM 可能不会在向最终用户销售的计算机上指定此密钥。
