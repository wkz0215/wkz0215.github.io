wbadmin enable backup

  • 项目
  • 09/21/2022

本文内容

创建并启用每日备份计划或修改现有备份计划。 未指定参数，会显示当前计划的备份设置。

若要使用此命令配置或修改每日备份计划，必须是 备份操作员 组或 管理员 组的成员。 此外，必须通过右键单击命令提示符，然后选择“以管理员身份运行”，从提升的命令提示符运行 wbadmin。

若要查看磁盘的磁盘标识符值，请运行 wbadmin get disks 命令。

语法

wbadmin enable backup [-addtarget:<BackupTarget>] [-removetarget:<BackupTarget>] [-schedule:<TimeToRunBackup>] [-include:<VolumesToInclude>] [-nonRecurseInclude:<ItemsToInclude>] [-exclude:<ItemsToExclude>] [-nonRecurseExclude:<ItemsToExclude>][-systemState] [-hyperv:<HyperVComponentsToExclude>] [-allCritical] [-systemState] [-vssFull | -vssCopy] [-user:<UserName>] [-password:<Password>] [-allowDeleteOldBackups]  [-quiet]

参数

参数 说明
-addtarget 指定备份的存储位置。 要求将位置指定为磁盘、卷或通用命名约定， (UNC) 路径指向远程共享文件夹 (\\<servername>\<sharename>) 。 默认情况下，备份将保存在： \\<servername>\<sharename> WindowsImageBackup <ComputerBackedUp> 如果指定磁盘，将在使用之前格式化磁盘，并且永久擦除磁盘上的任何现有数据。 如果指定共享文件夹，则无法添加更多位置。 一次只能将一个共享文件夹指定为存储位置。

重要： 如果将备份保存到远程共享文件夹，则使用同一文件夹再次备份同一台计算机时，将覆盖该备份。 此外，如果备份操作失败，则最终可能没有备份，因为旧备份将被覆盖，但较新的备份将不可用。 可以通过在远程共享文件夹中创建子文件夹来组织备份来避免这种情况。 如果执行此操作，子文件夹需要父文件夹的空间两倍。

单个命令中只能指定一个位置。 可以通过再次运行命令来添加多个卷和磁盘备份存储位置。
-removetarget 指定要从现有备份计划中删除的存储位置。 要求将位置指定为磁盘标识符。
-schedule 指定创建备份的时间，格式为 HH：MM 和逗号分隔。
-include 指定要包含在备份中的项的逗号分隔列表。 可以包含多个文件、文件夹或卷。 可以使用卷驱动器号、卷装入点或基于 GUID 的卷名称指定卷路径。 如果使用基于 GUID 的卷名称，它应以反斜杠 () \ 结尾。 指定文件的路径时，可以在文件名中使用通配符 (*) 。
-nonRecurseInclude 指定要包含在备份中的项的非递归逗号分隔列表。 可以包含多个文件、文件夹或卷。 可以使用卷驱动器号、卷装入点或基于 GUID 的卷名称指定卷路径。 如果使用基于 GUID 的卷名称，它应以反斜杠 () \ 结尾。 指定文件的路径时，可以在文件名中使用通配符 (*) 。 仅当使用 -backupTarget 参数时才应使用。
-exclude 指定要从备份中排除的项的逗号分隔列表。 可以排除文件、文件夹或卷。 可以使用卷驱动器号、卷装入点或基于 GUID 的卷名称指定卷路径。 如果使用基于 GUID 的卷名称，它应以反斜杠 () \ 结尾。 指定文件的路径时，可以在文件名中使用通配符 (*) 。
-nonRecurseExclude 指定要从备份中排除的项的非递归逗号分隔列表。 可以排除文件、文件夹或卷。 可以使用卷驱动器号、卷装入点或基于 GUID 的卷名称指定卷路径。 如果使用基于 GUID 的卷名称，它应以反斜杠 () \ 结尾。 指定文件的路径时，可以在文件名中使用通配符 (*) 。
-hyperv 指定要包含在备份中的组件的逗号分隔列表。 标识符可以是组件名称或组件 GUID (，) 不带大括号。
-systemState 除了使用 -include 参数指定的任何其他项之外，还创建包含系统状态的备份。 系统状态包含启动文件 (Boot.ini、NDTLDR、NTDetect.com) 、Windows 注册表（包括 COM 设置、SYSVOL (组策略和登录脚本) 、Active Directory 和 NTDS）。域控制器上的 DIT，如果安装了证书服务，则为证书存储。 如果服务器安装了 Web 服务器角色，则包含 IIS 元目录。 如果服务器是群集的一部分，则还包括群集服务信息。
-allCritical 指定包含操作系统状态的所有关键卷 (卷) 包含在备份中。 如果要为完整系统或系统状态恢复创建备份，此参数非常有用。 仅当指定 -backupTarget 时，才应使用它;否则，命令将失败。 可与 -include 选项一起使用。

提示： 关键卷备份的目标卷可以是本地驱动器，但它不能是备份中包含的任何卷。
-vssFull 使用卷影复制服务 (VSS) 执行完全备份。 将备份所有文件，更新每个文件的历史记录以反映其已备份，并且以前的备份日志可能会被截断。 如果未使用此参数， 则 wbadmin start backup 命令会进行复制备份，但要备份的文件历史记录不会更新。

谨慎：如果使用除Windows服务器备份以外的产品来备份当前备份中包含的卷上的应用，请不要使用此参数。 这样做可能会中断其他备份产品正在创建的增量备份、差异备份或其他类型的备份，因为它们依赖的历史记录来确定备份可能丢失的数据量，并且可能会不必要地执行完整备份。
-vssCopy 使用 VSS 执行复制备份。 所有文件都已备份，但要备份的文件的历史记录不会更新，因此可以保留更改、删除等文件以及任何应用程序日志文件的所有信息。 使用此类型的备份不会影响可能与此复制备份无关的增量备份和差异备份序列。 这是默认值。

警告： 备份副本不能用于增量备份或差异备份或还原。
-user 指定具有备份存储目标写入权限的用户 (（如果它是远程共享文件夹) ）。 用户必须是正在备份的计算机上的 管理员 或 备份操作员 组的成员。
-password 指定参数 -user 提供的用户名的密码。
-allowDeleteOldBackups 覆盖在升级计算机之前所做的任何备份。
-quiet 在不提示用户的情况下运行该命令。

示例

若要在上午 9：00 和下午 6：00 计划硬盘驱动器 E：、D：\mountpoint 以及 \\?\Volume{cc566d14-44a0-11d9-9d93-806e6f6e6963}\将文件保存到名为 DiskID 的磁盘，请键入：

wbadmin enable backup -addtarget:DiskID -schedule:09:00,18:00 -include:E:,D:\mountpoint,\\?\Volume{cc566d14-44a0-11d9-9d93-806e6f6e6963}\

若要在上午 12：00 和下午 7：00 安排 D：\documents 文件夹的每日备份到网络位置 \\backupshare\backup1，请使用 备份操作员的网络凭据 Aaren Ekelund (aekel) ，其密码为 $3hM9^5lp ，并且是域 CONTOSOEAST 的成员，用于对网络共享的访问进行身份验证，键入：

wbadmin enable backup –addtarget:\\backupshare\backup1 –include: D:\documents –user:CONTOSOEAST\aekel –password:$3hM9^5lp –schedule:00:00,19:00

若要计划卷 T： 和 D：\documents 文件夹在上午 1：00 以驱动器 H：，不包括文件夹 d:\documents\~tmp，并使用卷影复制服务执行完整备份，请键入：

wbadmin enable backup –addtarget:H: –include T:,D:\documents –exclude D:\documents\~tmp –vssfull –schedule:01:00
