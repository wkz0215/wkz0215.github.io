shutdown

  • 项目
  • 09/21/2022

本文内容

使你能够一次关闭或重启本地或远程计算机。

语法

shutdown [/i | /l | /s | /sg | /r | /g | /a | /p | /h | /e | /o] [/hybrid] [/fw] [/f] [/m \\computer][/t xxx][/d [p|u:]xx:yy [/c "comment"]]

参数

参数 说明
/i 显示 “远程关闭 ”框。 /i 选项必须是命令后面的第一个参数。 如果指定 了 /i ，则忽略所有其他选项。
/l 立即注销当前用户，且没有超时期限。 不能将 /l 与 /m 或 /t 一起使用。
/s 关闭计算机。
/sg 关闭计算机。 在下一次启动时，如果启用了 自动重启登录 ，设备会根据最后一个交互式用户自动登录和锁定。 登录后，它会重启任何已注册的应用程序。
/r 关闭后重新启动计算机。
/g 关闭计算机。 在下一次重启时，如果启用了 自动重启登录 ，设备将根据最后一个交互式用户自动登录和锁定。 登录后，它会重启任何已注册的应用程序。
/a 中止系统关闭。 仅在超时期间有效。 若要使用 /a，还必须使用 /m 选项。
/p 关闭本地计算机， (不是远程计算机) ，没有超时期限或警告。 只能对 /d 或 /f 使用 /p。 如果计算机不支持关闭电源功能，则当你使用 /p 时，它将关闭，但对计算机的电源将保持打开状态。
/h 如果启用了休眠，将本地计算机置于休眠状态。 /f 开关可用于 /h 开关。
混合 关闭设备并准备快速启动。 此选项必须与 /s 选项一起使用。
/fw 将此选项与关闭选项组合在一起会导致下一次重启转到固件用户界面。
/e 使你能够记录目标计算机上意外关闭的原因。
/o 转到 “高级启动选项” 菜单并重启设备。 此选项必须与 /r 选项一起使用。
/f 强制运行应用程序关闭而不警告用户。
谨慎： 使用 /f 选项可能会导致丢失未保存的数据。
/m \\<computername> 指定目标计算机。 不能与 /l 选项一起使用。
/t <xxx> 将关闭之前的超时时间段设置为 xxx 秒。 有效范围为 0-315360000 (10 年) ，默认值为 30。 如果超时期限大于 0，则 表示 /f 参数。
/d [p | u:]<XX>:<YY> 列出系统重启或关闭的原因。 支持的参数值为：
  • p - 指示计划重启或关闭。
  • u - 指示原因是用户定义的。

    注意
    如果未指定 p 或 u ，则重启或关闭计划外。

  • xx - 指定正整数（小于 256) ） (主要原因数。
  • Yy 指定正整数（小于 65536) ） (次要原因数。
/c <comment> 让你可以对关闭原因作详细注释。 必须先使用 /d 选项提供原因，并且必须将注释括在引号中。 最多可使用 511 个字符。
/? 在命令提示符处显示帮助，包括本地计算机上定义的主要和次要原因的列表。

备注

  • 必须向用户分配 “关闭系统 ”用户权限，才能关闭正在使用 关闭 命令的本地或远程管理的计算机。

  • 用户必须是 管理员 组的成员，才能批注本地或远程管理的计算机意外关闭。 如果目标计算机已加入域， 则域管理员 组的成员可能能够执行此过程。 有关详细信息，请参阅：

    • 默认本地组

    • 默认组

  • 如果要一次关闭多个计算机，可以使用脚本调用每台计算机的 关闭 ，也可以使用 关闭/i 来显示 “远程关闭 ”框。

  • 如果指定主要和次要原因代码，则必须首先在计划使用原因的每台计算机上定义这些原因代码。 如果未在目标计算机上定义原因代码，则关闭事件跟踪器无法记录正确的原因文本。

  • 请记住，使用 p 参数指示计划关闭。 不使用 p 参数，指示关闭计划外。

    • 使用 p 参数，沿计划外关闭的原因代码导致关闭失败。

    • 不使用 p 参数，并且只提供计划关闭的原因代码，也会导致关闭失败。

示例

若要强制应用关闭并在一分钟延迟后重启本地计算机，原因为 应用程序：维护 (计划内) 和注释“重新配置myapp.exe”，请键入：

shutdown /r /t 60 /c "Reconfiguring myapp.exe" /f /d p:4:1

若要使用与上一个示例相同的参数重启远程计算机 myremoteserver ，请键入：

shutdown /r /m \\myremoteserver /t 60 /c "Reconfiguring myapp.exe" /f /d p:4:1
