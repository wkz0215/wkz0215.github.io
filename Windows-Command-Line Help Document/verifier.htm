verifier

  • 项目
  • 09/21/2022

本文内容

适用范围：Windows Server 2022、Windows Server 2019、Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

驱动程序验证程序监视 Windows 内核模式驱动程序和图形驱动程序，目的是检测可能损坏系统的非法函数调用或操作。 驱动程序验证程序可以将Windows驱动程序设置为各种压力和测试，以查找不当行为。 可以配置要运行的测试，这允许你通过沉重的压力负载或更简化的测试来放置驱动程序。 还可以同时在多个驱动程序或一次一个驱动程序上运行驱动程序验证程序。

重要

必须位于计算机上的“管理员”组中才能使用驱动程序验证程序。 运行驱动程序验证程序可能会导致计算机崩溃，因此你应仅在用于测试和调试的计算机上运行此实用工具。

语法

verifier /standard /all
verifier /standard /driver NAME [NAME ...]
verifier /flags <options> /all
verifier /flags <options> /driver NAME [NAME ...]
verifier /rules [OPTION ...]
verifier /query
verifier /querysettings
verifier /bootmode [persistent | disableafterfail | oneboot]
verifier /reset
verifier /faults [Probability] [PoolTags] [Applications] [DelayMins]
verifier /faultssystematic [OPTION ...]
verifier /log LOG_FILE_NAME [/interval SECONDS]
verifier /volatile /flags <options>
verifier /volatile /adddriver NAME [NAME ...]
verifier /volatile /removedriver NAME [NAME ...]
verifier /volatile /faults [Probability] [PoolTags] [Applications] [DelayMins]
verifier /domain <types> <options> /driver ... [/logging | /livedump]
verifier /logging
verifier /livedump
verifier /?
verifier /help

参数

参数 说明
/all 指示驱动程序验证程序实用工具在下一次启动后验证所有已安装的驱动程序。
/bootmode [persistent | disableafterfail | oneboot | resetonunusualshutdown] 控制重新启动后是否启用驱动程序验证程序实用工具的设置。 若要设置或更改此选项，必须重新启动计算机。 可用的模式如下：
  • persistent - 确保驱动程序验证程序设置保留 (在许多重新启动期间保持有效状态) 。 这是默认设置。
  • disableafterfail - 如果Windows无法启动，此设置将禁用驱动程序验证程序实用工具以供后续重新启动。
  • oneboot - 仅在下次计算机启动时启用驱动程序验证程序设置。 驱动程序验证程序实用工具在后续重新启动时处于禁用状态。
  • resetonunusualshutdown - 驱动程序验证程序实用工具将持续到发生异常关闭。 可以使用其缩写“rous”。
/driver <driverlist> 指定将验证的一个或多个驱动程序。 driverlist 参数是按二进制名称（例如driver.sys）的驱动程序列表。 使用空格分隔每个驱动程序名称。 不支持通配符值，例如 n*.sys。
/driver.exclude <driverlist> 指定将从验证中排除的一个或多个驱动程序。 仅当选择所有驱动程序进行验证时，此参数才适用。 driverlist 参数是按二进制名称（例如driver.sys）的驱动程序列表。 使用空格分隔每个驱动程序名称。 不支持通配符值，例如 n*.sys。
/faults 在驱动程序验证程序实用工具中启用 低资源模拟 功能。 可以使用 /faults/flags 0x4代替 。 但是，不能与 /faults 子参数一/flags 0x4起使用。 可以使用 /faults 参数的以下子参数来配置低资源模拟：
  • Probability - 指定驱动程序验证程序实用工具将失败给定分配的概率。 在十进制或十六进制) 中键入数字 (，以表示驱动程序验证程序实用工具将失败分配的 10，000 次机会数。 默认值 600 表示 600/10000 或 6%。
  • 池标记 - 限制驱动程序验证程序实用工具无法使用指定池标记进行分配的分配。 可以使用通配符 (*) 来表示多个池标记。 若要列出多个池标记，请使用空格分隔标记。 默认情况下，所有分配都可能会失败。
  • 应用程序 - 限制驱动程序验证程序实用工具无法为指定程序分配的分配。 键入可执行文件的名称。 若要列出程序，请使用空格分隔程序名称。 默认情况下，所有分配都可能会失败。
  • DelayMins - 指定启动后的分钟数，在此期间驱动程序验证程序实用工具不会故意失败任何分配。 此延迟允许驱动程序在测试开始之前加载和系统稳定。 在十进制或十六进制) 中键入数字 (。 默认值为 7 (分钟) 。
/faultssystematic 指定 系统低资源 模拟的选项。 使用 0x40000 标志选择 “系统低资源 模拟”选项。 可以使用以下选项：
  • enableboottime - 在计算机重新启动时启用故障注入。
  • disableboottime - 在计算机重启期间禁用错误注入， (这是默认设置) 。
  • recordboottime - 在计算机重启模式时启用故障注入。
  • resetboottime - 在计算机重新启动时禁用错误注入，并清除堆栈排除列表。
  • enableruntime - 动态启用故障注入。
  • disableruntime - 动态禁用错误注入。
  • recordruntime - 动态启用故障注入（如果模式）。
  • resetruntime - 动态禁用故障注入并清除以前出错的堆栈列表。
  • querystatistics - 显示当前的故障注入统计信息。
  • incrementcounter - 递增测试传递计数器，用于确定何时注入错误。
  • getstackid COUNTER - 检索指示的注入堆栈标识符。
  • excludeestack STACKID - 从故障注入中排除堆栈。
/flags <options> 在下一次重新启动后激活指定的选项。 可以使用十进制或十六进制 (以 0x 前缀) 格式输入此数字。 允许以下值的任何组合：
  • 值：1 位或0x1 (位 0) - 特殊池检查
  • 值：2 位或0x2 (位 1) - 强制 IRQL 检查
  • 值：4 位或0x4 (位 2) - 低资源模拟
  • 值：8 位或0x8 (位 3) - 池跟踪
  • 值：16 位或0x10 (位 4) - I/O 验证
  • 值：32 位或0x20 (位 5) - 死锁检测
  • 值：64 位或0x40 (位 6) - 增强的 I/O 验证。 选择 I/O 验证时，将自动激活此选项。
  • 值：128 位或0x80 (位 7 位) - DMA 验证
  • 值：256 位或0x100 (位 8 位) - 安全检查
  • 值：512 位或0x200 (位 9) - 强制挂起 I/O 请求
  • 值：1024 位或0x400 (位 10 位) - IRP 日志记录
  • 值：2048 位或0x800 (位 11 位) - 杂项检查
  • 值：8192 或0x2000 (位 13 位) - 堆栈的固定 MDL 检查
  • 值：16384 或0x4000 (位 14) - 驱动程序的固定 MDL 检查
  • 值：32768 或 0x8000 (位 15 位) - Power Framework 延迟模糊处理
  • 值：65536 或0x10000 (位 16) - 端口/微型端口接口检查
  • 值：131072位或0x20000 (位 17 位) - DDI 符合性检查
  • 值：262144位或0x40000 (位 18 位) - 系统低资源模拟
  • 值：524288位或0x80000 (位 19 位) - DDI 符合性检查 (其他)
  • 值：2097152位或0x200000 (位 21 位) - NDIS/WIFI 验证
  • 值：8388608位或0x800000 (位 23) - 内核同步延迟模糊
  • 值：16777216位或0x1000000 (位 24) - VM 交换机验证
  • 值：33554432或0x2000000 (位 25) - 代码完整性检查。 不能使用此方法激活 SCSI 验证或 Storport 验证选项。 有关详细信息，请参阅 SCSI 验证 和 Storport 验证。
/flags <volatileoptions> 指定在不重新启动的情况下立即更改的驱动程序验证程序实用工具选项。此数字可以采用十进制或十六进制 (输入，其前缀为 0x) 格式。 允许以下值的任意组合：
  • 值：1 或0x1 (位 0) - 特殊池
  • 值：2 位或0x2 (位 1) - 强制 IRQL 检查
  • 值：4 或0x4 (位 2) - 低资源模拟
<probability> 指定故障注入概率的介于 1 和 10，000 之间的数字。 例如，指定 100 表示故障注入概率为 1% (100/10，000) 。

如果未指定此参数，则使用默认概率 6%。
<tags> 指定将注入错误（用空格字符分隔）的池标记。 如果未指定此参数，则任何池分配都可以注入故障。
<apps> 指定将注入错误的应用的图像文件名，用空格字符分隔。 如果未指定此参数，则可以在任何应用程序中进行低资源模拟。
<minutes> 一个正数，指定重新启动后的时间段长度（以分钟为单位），在此期间不会发生错误注入。 如果未指定此参数，则使用默认长度 8 分钟 。
/iolevel <level> 指定 I/O 验证级别。 [level] 的值可以是 1 - 启用级别 1 I/O 验证 (默认) 或 2 - 启用级别 1 I/O 验证和级别 2 I/O 验证。 如果未使用 /flags 0x10) 启用 I/O 验证 (，则 忽略 /iolevel 。
/log <logfilename> [/intervalseconds] 使用指定名称创建日志文件。 驱动程序验证程序实用工具根据可以选择设置的间隔定期将统计信息写入此文件。 默认间隔为 30 秒。

如果在命令行键入验证程序 /log 命令，则命令提示符不会返回。 若要关闭日志文件并返回提示，请使用 Ctrl+C 键。 重新启动后，若要创建日志，必须再次提交验证程序 /log 命令。
/rules <option> 可禁用的规则选项，包括：
  • 查询 - 显示可控制规则的当前状态。
  • reset - 将所有规则重置为其默认状态。
  • 默认 ID - 将规则 ID 设置为其默认状态。 对于支持的规则，规则 ID 是 Bug 检查0xC4 (DRIVER_VERIFIER_DETECTED_VIOLATION) 参数 1 值。
  • 禁用 ID - 禁用指定的规则 ID。 对于支持的规则，规则 ID 是 Bug 检查0xC4 (DRIVER_VERIFIER_DETECTED_VIOLATION) 参数 1 值。
/standard 在下一次重启后激活“标准”或默认驱动程序验证程序选项。 标准选项包括特殊池、强制 IRQL 检查、池跟踪、I/O 验证、死锁检测、DMA 验证、安全检查、杂项检查和 DDI 符合性检查。 这等效于 /flags 0x209BB。

[!注意] 从 1803 之后的 Windows 10 版本中开始，使用/flags 0x209BB将不再自动启用 WDF 验证。 使用 /standard 语法启用标准选项，并包含 WDF 验证。
/volatile 在不重新启动计算机的情况下更改设置。 易失性设置立即生效。

可以将 /volatile 参数与 /flags 参数结合使用，以便在不重新启动的情况下启用和禁用某些选项。 还可以对 /adddriver 和 /removedriver 参数使用 /volatile 来启动或停止驱动程序的验证，而无需重新启动，即使驱动程序验证程序实用工具未运行。 有关详细信息，请参阅使用易失性设置。
/adddriver <volatiledriverlist> 从易失性设置中删除指定的驱动程序。 若要指定多个驱动程序，请列出其名称，用空格分隔。 不支持通配符值（如 n.sys）。
/removedriver <volatiledriverlist>
/reset 清除所有驱动程序验证程序实用工具设置。 下次重启后，不会验证任何驱动程序。
/querysettings 显示将激活的选项摘要，以及将在下一次启动后验证的驱动程序。 显示不包括使用 /volatile 参数添加的驱动程序和选项。 有关查看这些设置的其他方法，请参阅查看驱动程序验证程序设置。
/query 显示驱动程序验证程序实用工具当前活动的摘要。 显示中的 “级别 ”字段是使用 /volatile 参数设置的选项的十六进制值。 有关每个统计信息的说明，请参阅 监视全局计数器 和 监视单个计数器。
/domain <types> <options> 控制验证程序扩展设置。 支持以下验证程序扩展类型：
  • wdm - 为 WDM 驱动程序启用验证程序扩展。
  • ndis - 为网络驱动程序启用验证程序扩展。
  • ks - 为内核模式流驱动程序启用验证程序扩展。
  • 音频 - 为音频驱动程序启用验证程序扩展。
. 支持以下扩展选项：
  • rules.default - 为所选验证程序扩展启用默认验证规则。
  • rules.all - 为所选验证程序扩展启用所有验证规则。
/logging 为所选验证程序扩展检测到的违反规则启用日志记录。
/livedump 为所选验证程序扩展检测到的违反规则启用实时内存转储收集。
/? 显示命令行帮助。

返回代码

驱动程序验证程序运行后，将返回以下值：

  • 0：EXIT_CODE_SUCCESS

  • 1：EXIT_CODE_ERROR

  • 2：EXIT_CODE_REBOOT_NEEDED

注解

  • 可以将 /volatile 参数与某些驱动程序验证程序实用工具 /flags 选项和 /standard 配合使用。 不能将 /volatile 用于 DDI 符合性检查、Power Framework 延迟模糊、Storport 验证或 SCSI 验证的 /flags 选项。 有关详细信息，请参阅使用易失性设置。
