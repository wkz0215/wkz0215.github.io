at

  • 项目
  • 09/21/2022

本文内容

适用范围：Windows Server 2022、Windows Server 2019、Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

计划命令和程序，以在指定的时间和日期在计算机上运行。 只能在计划 服务 运行时使用 。 在没有参数的情况下使用 ，在 列表中 列出计划的命令。 你必须是本地 Administrators 组的成员才能运行此命令。

语法

at [\computername] [[id] [/delete] | /delete [/yes]]
at [\computername] <time> [/interactive] [/every:date[,...] | /next:date[,...]] <command>

参数

参数 说明
\<computername\> 指定远程计算机。 如果省略此参数 ，则 在本地 计算机上计划命令和程序。
<id> 指定分配给计划命令的标识号。
/delete 取消计划的命令。 如果省略 ID，将取消计算机上所有计划的命令。
/yes 删除计划事件时，系统的所有查询的回答为"是"。
<time> 指定运行命令的时间。 time 表示为 Hours：Minutes（以 24 小时表示法表示 (，即午夜 00：00 () 23：59) 。
交互式 允许命令与命令运行时登录的用户的桌面交互。
每： 在 每周 或每月的指定日期（例如 (星期四或每月的第三天）运行命令) 。
<date> 指定运行命令的日期。 可以指定一周中的一天或多天 (即键入 M、T、W、Th、F、S、Su) 或一个或多个月中的一天 (即键入 1 到 31) 。 用逗号分隔多个日期条目。 如果省略 日期， 则 使用 当月的当前日期。
下： 在 每天 的下一个匹配项上运行 (例如，下一个星期四) 。
<command> 指定 Windows 命令、 (程序 (（即 .exe 或 .com 文件) ）或批处理程序 (，即要运行的 .bat 或 .cmd) 文件。 当命令需要路径作为参数时，请使用绝对 (路径，即以驱动器号开头的整个) 。 如果命令位于远程计算机上，请为服务器和共享 (UNC) 指定通用命名约定，而不是远程驱动器号。
/? 在命令提示符下显示帮助。

注解

  • 此命令在运行命令之前不会自动cmd.exe加载。 如果未运行可执行文件 (.exe) ，则必须在命令cmd.exe显式加载文件，如下所示：

    cmd /c dir > c:\test.out

  • 如果在没有命令行选项的情况下使用此命令，则计划任务将显示在格式类似于以下内容的表中：

    Status  ID   Day        time        Command Line
OK      1    Each F     4:30 PM     net send group leads status due
OK      2    Each M     12:00 AM    chkstor > check.file
OK      3    Each F     11:59 PM    backup2.bat

  • 如果使用此命令将标识 (ID) ，则只有单个条目的信息以类似于以下格式显示：

    Task ID: 1
Status: OK
Schedule: Each  F
Time of Day: 4:30 PM
Command: net send group leads status due

  • 计划命令（尤其是具有命令行选项的命令）后，请通过键入 不带任何命令行选项的 来 检查命令语法是否正确。 如果"命令行" 列中 的信息错误，请删除命令，然后重新键入该命令。 如果仍然不正确，则使用更少的命令行选项重新键入命令。

  • 使用 在 中 计划的命令 作为后台进程运行。 输出不显示在计算机屏幕上。 若要将输出重定向到文件，请使用重定向符号 >。 如果将输出重定向到文件，^则需要在重定向符号之前使用转义符号，不管是在命令行使用 还是批处理文件中使用 。^ 例如，若要将输出重定向到 output.txt，请键入：

    at 14:45 c:\test.bat ^>c:\output.txt

    执行命令的当前目录是 systemroot 文件夹。

  • 如果在计划运行命令后更改系统时间，请通过键入 不带命令行选项的 来将计划程序与修改后的系统时间同步。

  • 计划的命令存储在注册表中。 因此，如果重启计划服务，不会丢失计划任务。

  • 请勿对访问网络的计划作业使用重定向驱动器。 计划服务可能无法访问重定向的驱动器，或者，如果其他用户在计划任务运行时登录，则重定向的驱动器可能不存在。 请改为对计划作业使用 UNC 路径。 例如：

    at 1:00pm my_backup \\server\share

    请勿使用以下语法，其中 x： 是用户建立的连接：

    at 1:00pm my_backup x:

    如果计划使用驱动器号连接到共享目录的 at 命令，请包含 at 命令，以在使用完驱动器后断开驱动器连接。 如果驱动器未断开连接，则指定的驱动器号在命令提示符下不可用。

  • 默认情况下，使用此命令计划的任务将在 72 小时后停止。 可以修改注册表以更改此默认值。

    修改注册表

    注意

    不正确地编辑注册表可能会对系统造成严重损坏。 在更改注册表之前，应备份计算机上任何有价值的数据。

    1. 启动注册表编辑器 (regedit.exe) 。

    2. 在注册表中找到并单击以下项： HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Schedule

    3. 在" 编辑" 菜单上，单击" 添加值"，然后添加以下注册表值：

      • 值名称。 atTaskMaxHours

      • 数据类型。 reg_DWOrd

      • 基数。 小数

      • 值数据： 0. "值数据"字段中的值 0 表示没有限制，并且不会停止。 从 1 到 99 的值指示小时数。

  • 可以使用"计划任务"文件夹查看或修改使用此命令创建的任务的设置。 使用此命令计划任务时，任务将列在"计划任务"文件夹中，其名称如下所示：at3478。 但是，如果通过"计划任务"文件夹修改任务，该任务将升级到常规计划任务。 任务不再对 at 命令可见，并且 at 帐户设置不再适用于它。 必须显式输入任务的用户帐户和密码。

示例

若要显示市场营销服务器上计划的命令列表，请键入：

at \\marketing

若要详细了解 Corp 服务器上标识号为 3 的命令，请键入：

at \\corp 3

若要将 net share 命令安排在上午 8：00 在 Corp 服务器上运行，将列表重定向到维护服务器，请键入"报表共享目录"和 Corp.txt 文件：

at \\corp 08:00 cmd /c net share reports=d:\marketing\reports >> \\maintenance\reports\corp.txt

若要将市场营销服务器的硬盘驱动器备份到每五天午夜的磁带驱动器，请创建名为 Archive.cmd 的批处理程序，其中包含备份命令，然后计划批处理程序运行，键入：

at \\marketing 00:00 /every:5,10,15,20,25,30 archive

若要取消当前服务器上计划的所有命令，请清除 "按 计划" 信息，如下所示：

at /delete

若要运行不是可执行文件 (.exe) 的命令，在命令之前使用 cmd /c 加载cmd.exe如下所示：

cmd /c dir > c:\test.out
