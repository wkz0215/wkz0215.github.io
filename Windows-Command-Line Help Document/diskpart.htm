diskpart

  • 项目
  • 10/05/2022

本文内容

适用于：Windows Server 2022、Windows 10、Windows 8.1、Windows 8、Windows 7、Windows Server 2019、Windows Server 2016、Windows Server 2012 R2、Windows Server 2012 和 Windows Server 2008 R2，Windows Server 2008

diskpart 命令解释器可帮助管理计算机的驱动器 (磁盘、分区、卷或虚拟硬盘) 。

在使用 diskpart 命令之前，必须先列出，然后选择一个对象来使其具有焦点。 对象具有焦点后，键入的任何 diskpart 命令都将对该对象执行操作。

确定焦点

选择对象时，焦点一直停留在该对象上，直至选择其他对象。 例如，如果在磁盘 0 上设置了焦点，并且选择磁盘 2 上的卷 8，则焦点将从磁盘 0 转移到磁盘 2，卷 8。

某些命令会自动更改焦点。 例如，创建新分区时，焦点会自动切换到新分区。

只能将焦点放在所选磁盘上的分区上。 分区具有焦点后，如果有任何) 也具有焦点，则相关卷 (。 卷具有焦点后，如果卷映射到单个特定分区，则相关磁盘和分区也具有焦点。 如果情况并非如此，则焦点位于磁盘上，分区将丢失。

语法

若要启动 diskpart 命令解释器，请在命令提示符下键入：

diskpart <parameter>

重要

必须位于本地 管理员 组或具有类似权限的组才能运行 diskpart。

offline 将联机磁盘或卷带到脱机状态。

示例

若要查看可用磁盘 () ，请运行 列表磁盘：

list disk

若要选择磁盘，请运行 选择磁盘 ，然后运行磁盘编号。 例如：

select disk 1

在使用磁盘 1 之前，需要通过运行 创建分区主数据库来创建分区：

create partition primary

最后，可以通过运行 格式 fs=ntfs label=Backup quick ，以磁盘 1 到 NTFS 的快速格式执行磁盘 1 到 NTFS 的格式，如下所示：
